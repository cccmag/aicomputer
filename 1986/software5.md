# 物件導向程式設計的工具化

## 超越 C++

1986 年，物件導向程式設計（OOP）開始從學術概念轉向實用工具——這不僅是 C++ 的功勞，還有多種物件導向工具和環境的出現。

這一年值得注意的發展包括：

**Object Pascal**：Apple 在 Macintosh 上推出了 Object Pascal——對標準 Pascal 的物件導向擴展。它在 MacApp（Apple 的應用程式框架）中被廣泛使用。

**Smalltalk 商業化**：ParcPlace Systems——由 Xerox PARC 的研究員創立——開始將 Smalltalk-80 作為商業產品銷售。Smalltalk 是純粹的物件導向語言，一切皆物件。

**Flavors 與 CLOS**：Lisp 機器上的物件導向系統 Flavors 和 Common Lisp Object System（CLOS）開始標準化，影響了後來 Python 和 Ruby 的設計。

## 物件導向的實用化

1986 年的重要趨勢是物件導向從「設計哲學」走向「工程工具」：

**類別庫**：開發人員開始建立可重複使用的類別庫——這是現代框架和套件管理系統的前身。

**GUI 工具包**：MacApp 等物件導向框架讓 Macintosh 應用程式的開發效率大幅提升。

**設計方法論**：物件導向分析與設計（OOAD）的概念開始萌芽，為日後的 UML 和設計模式奠定了基礎。

## 影響

1986 年的 OOP 工具化為 1990 年代物件導向的全面普及做了準備。當 Java 在 1995 年推出時，物件導向已經有了十年的實戰經驗和成熟的工具支援。
