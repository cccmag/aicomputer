# 歷史穿梭：1987 年的種子

## RISC 的傳承：從 IBM 801 到 ARM

1987 年 SPARC 處理器的推出是 RISC 架構商業化的重要一步。這條路線的簡史：

**IBM 801 (1975, 研究)** → **Berkeley RISC (1981)** → **SPARC (1987)** → **ARM (1985)** → **PowerPC (1991)** → **MIPS (1985)**

今天，ARM 架構統治了行動裝置市場（全球超過 95% 的智慧型手機使用 ARM 核心），蘋果也從 Intel x86 轉向了 ARM（M1 系列晶片）。RISC 從一個實驗室概念成長為計算機產業的主導力量。

1987 年的圖靈獎得主 John Cocke 如果在天有靈，看到他的 RISC 概念如何主宰了現代計算，應該會感到欣慰。

## 從 HyperCard 到 WWW

HyperCard 的超文本概念（1987）與兩年後 Tim Berners-Lee 的 WWW 提案（1989）之間存在一條明確的傳承：

**Memex (1945, Bush)** → **NLS (1968, Engelbart)** → **Hypertext 概念 (1965, Nelson)** → **HyperCard (1987)** → **WWW (1989)**

Berners-Lee 曾表示，HyperCard 對他的影響在於證明了一般使用者能夠接受非線性的超文本資訊組織方式。雖然 WWW 的技術設計與 HyperCard 差異很大（分散式 vs 本地端、伺服器-客戶端 vs 單機），但 HyperCard 為 WWW 的社會接受度鋪平了道路。

## 微軟的 Windows 之路

Windows 的發展在 1987 年仍然不順利，但 2.0 版的基礎為後續的爆發奠定了基礎：

**Windows 1.0 (1985)** → **Windows 2.0 (1987)** → **Windows 3.0 (1990, 突破)** → **Windows 3.1 (1992)** → **Windows 95 (1995)**

Windows 2.0 引入的重疊視窗和 Windows/386 的多工能力，是 Windows 3.0 取得商業成功的必要前提。Apple 對 Windows 2.0 的訴訟雖然沒有公開改變結果，但促使 Microsoft 在後續版本中更加注意介面設計的差異化。

## Perl 與 CGI 時代

Larry Wall 的 Perl 在 1987 年 12 月問世時，沒有人預料到它將在 1990 年代成為網際網路的「膠帶」。

Perl 的文字處理能力和靈活性使其成為 CGI 程式設計的語言首選。早期網站的留言板、計數器、表單處理和內容管理系統，大部分是用 Perl 寫的。Perl 的動態語法也啟發了後來的 PHP、Python 和 Ruby。

## 多媒體 PC 的誕生

1987 年發生的幾件事——AdLib 音效卡、CD-ROM 遊戲、Macintosh II 彩色顯示——共同描繪了「多媒體 PC」的未來：

**文字模式 PC (1980-1985)** → **AdLib + VGA (1987)** → **SoundBlaster (1988)** → **多媒體 PC 標準 MPC (1990)**

當聲音、彩色圖像和大容量儲存整合在一起時，PC 從「生產力工具」轉變為「娛樂與創作平台」——這個轉變在 1990 年代中期隨著 CD-ROM 的普及而完成。

## 1987 年的時代脈絡

- **冷戰尾聲**：美蘇關係緩和，戈巴契夫的改革開放（Perestroika）影響了全球科技交流
- **消費電子擴張**：第一屆 SXSW 音樂節在德州舉行，預示了科技與文化的融合
- **PC 產業的分裂**：IBM 的封閉策略（PS/2 MCA）與開放陣營（EISA）的戰爭重新定義了 PC 產業格局
- **網路社群雛形**：Habitat 等專案開始探索線上社交的可能性
