# 歷史穿梭：1982 年的種子

## 從 Lotus 1-2-3 到雲端辦公室

Lotus 1-2-3 的「整合式軟體」概念是現代辦公室套裝軟體的始祖：

**Lotus 1-2-3 (1983)** → **Microsoft Excel (1985)/Word (1983)** → **Microsoft Office (1990)** → **Google Workspace (2006)** → **雲端協作**

Lotus 1-2-3 的重要性不僅在於其技術，更在於它證明了：將多個功能整合在一個一致的使用者介面中，比單獨使用多個獨立程式更有生產力。這個理念在四十年後仍然驅動著 Microsoft 365 和 Google Workspace。

## 從 C++ 到現代系統程式設計

C++ 在 1982 年的誕生開啟了物件導向系統程式設計的新時代：

**C with Classes (1982)** → **C++ (1985)** → **Java (1995)** → **C# (2000)** → **Rust (2010, 零成本抽象)**

C++ 的「零成本抽象」原則是其持久成功的關鍵：使用類別、虛擬函數、範本等高階抽象時，不付出不必要的執行效能代價。Rust 繼承了這個原則，將它推向了記憶體安全的極致。

## 從 TCP/IP 到全球網際網路

1982 年 TCP/IP 成為 ARPANET 標準，是網際網路從單一網路發展為「網路的網路」（internetwork）的關鍵時刻：

**TCP/IP 標準化 (1982)** → **NSFNET (1986)** → **WWW 發明 (1990)** → **Mosaic 瀏覽器 (1993)** → **商用網際網路 (1995)** → **全球 50 億用戶**

如果 1982 年美國國防部選擇了其他協定（如 X.25 或 OSI 協定族），今天的網路可能會大不相同。TCP/IP 的簡潔、開放、端到端原則（end-to-end principle）是其最終勝出的原因。

## 從 Compaq 到整個 PC 產業

Compaq 的逆向工程策略開啟了 PC 相容機時代：

**Compaq Portable (1982)** → **Dell (1984, 直銷模式)** → **無數 PC 製造商** → **Wintel 壟斷** → **Apple M1 晶片 (2020, 重新定義個人電腦)**

PC 相容機市場的爆炸式成長降低了個人電腦的價格，讓全球數億人擁有電腦——但同時也導致了利潤率的持續下滑。Compaq 最終在 2002 年被 HP 收購，結束了二十年的獨立發展。

## 從 Commodore 64 到家用遊戲主機

Commodore 64 的遊戲生態系是現代遊戲產業的雛形：

**C64 (1982)** → **Nintendo Entertainment System (1985)** → **PC 遊戲 (1990s)** → **PlayStation (1994)** → **手機遊戲 (2000s)**

C64 說明了「硬體規格」不是成功的唯一因素——它優異的圖形和音效晶片讓開發者能夠創造出令人驚豔的遊戲。這個教訓——好的開發者工具和 API 比原始效能更重要——被 Nintendo、Sony 和 Apple 深刻理解。

## 1982 年的時代脈絡

- **IBM PC 相容機的誕生**：Compaq 的成功證明了開放架構的雙面性——對新進者是大門，對 IBM 是陷阱
- **日本經濟的頂峰**：第五代電腦計畫反映了日本對科技領導地位的渴望。日本在半導體、汽車和消費電子領域已超越美國
- **AI 的冬天即將來臨**：1982 年是專家系統熱潮的尾聲。FGCS 的邏輯程式設計路線在兩年後開始受到挑戰，而 1987 年的 AI 冬天即將終結這波熱潮
- **AT&T 的拆分**：1982 年美國司法部與 AT&T 達成和解，AT&T 同意拆分區域電話公司。這項決策降低了電話通訊成本，間接促進了數據機網路的普及
