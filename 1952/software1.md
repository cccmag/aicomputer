# 葛麗絲·霍普的 A-0 編譯器 — 軟體工程革命的起點

一九五二年，葛麗絲·霍普（Grace Hopper）在 Remington Rand 的 UNIVAC 團隊中完成了 A-0 系統（Arithmetic Language version 0）。A-0 是世界上第一個編譯器，它允許程式設計師使用符號形式的數學表達式編寫程式，然後自動將其轉換為機器碼。

霍普的靈感來自於她對程式設計效率的深刻不滿。在 UNIVAC I 上編寫程式的過程極為繁瑣：程式設計師需要將所有指令和資料以二進位或八進位形式寫在編碼表上，然後由操作員手工打孔。一個小錯誤可能導致整個程式需要從頭除錯數小時。霍普認為，計算機本身應該能夠承擔部分程式設計工作。

A-0 的核心概念是副程式（subroutine）庫。霍普預先將常用的數學函數——如正弦、餘弦、平方根等——編寫為標準的機器碼程式，並存放在磁帶上。當程式設計師在程式中需要用到這些函數時，只需寫一個簡短的符號調用，A-0 就會自動從函式庫中找到相應的副程式，將它連結到主程式中。

這個「副程式連結」的概念在今天是程式設計的常識，但在一九五二年卻是革命性的。當時的程式設計師習慣於每次都從頭編寫所有程式碼。霍普的方案不僅減少了重複勞動，更重要的是建立了程式碼複用的基本模式。

A-0 的運作方式是：程式設計師將原始碼寫在打孔卡片上——使用一種接近數學符號的語法。然後將卡片送入 UNIVAC I，A-0 編譯器讀取卡片，執行三個階段：記錄（將副程式從磁帶載入）、選擇（解析符號引用並決定哪個副程式需要被調用）、以及連結（將副程式的機器碼與主程式整合）。最終輸出的是一疊可在 UNIVAC I 上直接執行的卡片。

霍普後來回憶說，她將 A-0 稱為「編譯器」時，同事們的反應是懷疑和不解。「計算機只會做算術和邏輯，不可能寫程式，」他們說。但霍普堅持這個想法，並在後來的 A-1、A-2 系統中不斷改進。A-2 後來被分發給 UNIVAC 用戶，成為第一個被廣泛使用的編譯器。

A-0 的遺產既深遠又具體。霍普在一九五二年發展的副程式連結概念，後來被所有的程式語言採用——從 FORTRAN 和 COBOL 到 C 和 Java。霍普對「編譯器」這個詞的創造，也定義了一個全新的軟體類別。更廣泛地說，A-0 證明了「軟體可以自動化軟體的生產」——這個自我指涉的洞見，是整個軟體工程學科的哲學基礎。

葛麗絲·霍普後來被稱為「COBOL 之母」，但她最根本的貢獻是 A-0 編譯器。一九五二年 A-0 的誕生，是人類探索「如何讓電腦更易於程式設計」這條道路上第一個、也是最關鍵的一步。
