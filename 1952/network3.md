# 貝爾實驗室資料通訊 — 數位訊號在類比網路上的傳遞

一九五二年，貝爾實驗室的工程師正在探索一個日益重要的問題：如何透過電話網路傳輸數位資料？雖然這個問題的實用性在當時還不明顯，但其答案將在二十年後促成數據機和電腦網路的誕生。

美國的電話網路在當時是世界上最大的機器系統，擁有數千萬部電話機和數百萬英里的線路。但這個網路是為類比語音訊號設計的——人聲轉換為連續變化的電壓，經過放大和交換，最終在另一端還原為聲音。數位資料是離散的脈衝（0 和 1），透過類比網路傳輸會遇到嚴重的訊號衰減和失真問題。

貝爾實驗室的研究人員嘗試了多種調變技術。最直接的方法是幅移鍵控（ASK）——用訊號振幅的高低來表示 0 和 1。但這種方法對雜訊極其敏感。另一種方法是頻移鍵控（FSK）——用不同的頻率來表示 0 和 1。FSK 的抗雜訊能力較好，但帶寬效率較低。

還有更根本的問題需要解決。電話交換機是為連續通話設計的——當一方掛斷時，線路才會釋放。數位通訊可能需要短暫的間歇，但交換機會將這些間歇理解為「通話結束」而切斷連接。為了解決這個問題，貝爾實驗室的工程師設計了「保持訊號」——一個在資料傳輸間隙持續發送的載波，以防止交換機切斷線路。

一九五二年，這些研究還處於概念階段。貝爾實驗室能夠在實驗室環境中實現低速（約每秒一百到兩百位元）的資料傳輸，但距離實用化的產品還有數年之遙。第一個商業數據機——Bell 101——要到一九五八年才推出。

儘管如此，這些早期研究確立了數位通訊的基本技術框架。它們定義了「調變」（modulation）、「解調」（demodulation）和「數據機」（modem）這些概念，並為後續的技術標準化奠定了基礎。沒有這些概念，後來的網際網路就不可能存在。

一九五二年貝爾實驗室的研究還揭示了一個更深層的問題：數位時代的通訊系統需要從頭設計，而非在類比系統上修修補補。這個認識在數十年後催生了全數位化的通訊網路——從 ISDN 到光纖、從 4G 到 5G。現在回頭看，一九五二年那些在實驗室裡研究「如何將 0 和 1 塞進電話線」的工程師，其實是在為一個還不存在的世界鋪設基礎設施。
