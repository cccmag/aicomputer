# UNIVAC Short Code — 第一部高階語言的實踐

一九五二年，UNIVAC I 上的 Short Code 仍然是少數程式設計師探索「更高層次的程式設計」的主要工具。Short Code 由約翰·莫奇利（John Mauchly）於一九四九年開發，是世界上第一個專門為電子計算機設計的高階語言。

Short Code 的語法設計體現了莫奇利對程式設計的看法：程式設計師應該能夠用接近數學的符號來表達計算邏輯。例如，要計算一個公式如 X = (A + B) × C，在機器語言中需要多條指令（載入、加法、乘法、儲存），而在 Short Code 中只是幾行簡短的符號表達式。

Short Code 的本質是一個直譯器——一支執行在 UNIVAC I 上的特殊程式，它讀取用 Short Code 語法編寫的偽碼並即時轉換為機器指令。這種執行方式與現代腳本語言的做法極為相似。Short Code 支援大約五十種操作，包括基本的算術運算、三角函數、對數函數和平方根等。

Short Code 的語法極為簡潔。運算元以變數名稱表示（如 X、Y、Z），操作以兩位數的數字代碼表示——例如「01」代表加法、「02」代表乘法、「03」代表除法。程式碼以一個特殊的「開始」符號為起點，以「結束」符號為終點。

然而，Short Code 的使用體驗遠非順暢。由於 UNIVAC I 的記憶體僅有 1000 個字，Short Code 直譯器本身就占據了大部分空間，留給使用者程式的空間極其有限。此外，直譯器每秒只能處理十到二十個符號指令——比等價的機器語言慢數十倍。

儘管有這些限制，Short Code 的使用者經驗驗證了一個關鍵假設：高階語言是可行的，程式設計師可以用更接近人類思考的方式與計算機溝通。這個經驗在兩年後對 IBM 的 FORTRAN 開發團隊產生了直接影響。

從 Short Code 到 FORTRAN 的演化並非簡單的線性過程，但兩者之間的傳承關係清楚可見。FORTRAN 將 Short Code 的直譯式執行改為編譯式執行，大幅提升了效能；將 Short Code 的數字操作碼改為英文關鍵字，提高了可讀性；將 Short Code 有限的操作集合擴展為完整的數學和邏輯運算系統。

一九五二年的 Short Code 雖然只有少數使用者，但它代表了一種思維方式的轉變：從「程式設計師適應機器」轉向「機器適應程式設計師」。這個轉變的影響在隨後七十年中持續擴大，並在二十一世紀的 Python、JavaScript 等語言中達到頂峰。
