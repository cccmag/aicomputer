# FORTRAN 專案在 IBM 啟動

一九五四年九月，IBM 的約翰·巴克斯（John Backus）正式啟動了一個名為「Formula Translation」的研發專案，簡稱 FORTRAN。這個為期三年的專案目標是創造一種程式語言，讓科學家與工程師能夠使用數學公式來表達計算邏輯，而不必學習電腦的機器語言。

巴克斯在提出這項構想時面臨了相當大的懷疑。當時的普遍觀點是：任何不是機器語言的東西都必然效率低下，無法與手工編寫的組合語言競爭。但巴克斯認為，隨著電腦硬體變得越來越複雜，程式設計的效率將成為瓶頸——如果科學家必須花費大量時間學習機器語言並手工編寫程式，電腦的實際使用價值就會受到限制。

巴克斯團隊最初的成員包括厄文·齊勒（Irving Ziller）與哈蘭·赫里克（Harlan Herrick），隨後羅伯特·尼爾森（Robert Nelson）等人加入。團隊的目標是為 IBM 704 電腦開發 FORTRAN 編譯器。選擇 704 作為目標平台是合理的——704 的磁芯記憶體與浮點運算能力使其成為當時最強大的科學電腦。

FORTRAN 專案的核心設計理念是「接近數學公式的語法」與「接近機器語言的效率」。團隊相信這兩者並不矛盾——透過適當的編譯器最佳化，FORTRAN 程式可以達到與手寫組合語言相當的執行效率。這個信念後來被證明是正確的。

FORTRAN 的語言設計在一九五四年到一九五五年間逐步成形。團隊確定了運算子優先級（先乘除後加減）、變數命名規則（以字母開頭）、陣列存取語法、DO 迴圈與 IF 條件判斷的基本結構。這些設計選擇後來被大多數程式語言所繼承。

FORTRAN 專案的啟動不僅是程式語言的誕生，也是編譯器設計這門學科的起點。巴克斯團隊在開發過程中發明了詞法分析、語法分析、中間碼生成與程式碼最佳化等一系列技術，這些技術至今仍然是編譯器設計的核心。
