# UNIVAC Short Code 直譯器與霍普的編譯器

一九五四年，UNIVAC 的程式設計工具正處於從機器語言向更高層次抽象過渡的時期。葛麗絲·霍普（Grace Hopper）在 UNIVAC 的工作，以及 Short Code 直譯器的發展，代表了這過渡時期的兩個重要方向。

Short Code 是由約翰·莫克（John Mauchly）與威廉·施密特（William Schmitt）於一九四九年為 UNIVAC I 開發的程式語言，被認為是第一部高階語言的嘗試。Short Code 允許程式設計師使用類似數學表達式的語法來撰寫程式，例如將「A = B + C」表示為「01 A 03 B 02 C」。

然而，Short Code 是直譯而非編譯的。程式碼在執行時由直譯器逐條讀取、解析並執行，而不是被預先轉換為機器碼。這使得 Short Code 的執行速度極其緩慢——比等效的機器語言慢數十倍到數百倍。因此，Short Code 主要用於教學與原型開發，而非實際的生產計算。

葛麗絲·霍普在 UNIVAC 開發了 A-0 系統（Arithmetic Language version 0），她稱之為「編譯器」。然而，A-0 與現代編譯器的概念有所不同。A-0 更像是一個連結器與載入器——它將預先編寫好的副程式（儲存在磁帶上）根據使用者的規格組合起來，產生可執行的機器碼。

霍普的關鍵創新在於她證明了程式可以用高於機器碼的層次來撰寫，並由機器自動轉換為可執行的形式。她後來領導了 FLOW-MATIC 語言的開發，這是第一部使用類似英文語法的程式語言，也是 COBOL 語言的主要前身。

UNIVAC 系統還使用了 C-10 指令集，這是一套設計精巧的六位元指令編碼系統。C-10 的每條指令包含一個四位元的操作碼與一個兩位元的修飾碼，能夠表示十六種操作與四種定址模式。C-10 的設計影響了後續 UNIVAC 系列的指令集架構，也為組合語言的發展提供了基礎。
