# Kubernetes 成為 CNCF 主導

## 容器編排的勝者

2017 年是 Kubernetes 鞏固其容器編排霸主地位的關鍵年份。雖然 Kubernetes 1.0 在 2015 年就發布了，但 2017 年見證了它從 Google 的主導專案轉變為整個雲端原生生態系統的核心。

這一年的標誌性事件包括：Docker 在 2017 年宣布原生支援 Kubernetes（Docker Enterprise Edition），這意味著 Docker 正式承認了 Kubernetes 在編排領域的勝利。Mesosphere（DC/OS）和 Docker Swarm 這兩個競爭對手的市場份額開始顯著下滑。

## CNCF 的擴張

2017 年，CNCF 的專案版圖大幅擴張。除了 Kubernetes 外，CNCF 還接納了 Prometheus（監控）、Envoy（代理）、CoreDNS（DNS）、containerd（容器執行時）、rkt（容器引擎）等關鍵專案。

Kubernetes 的 API 設計理念——宣告式 API、控制迴圈、自訂資源定義（CRD）——成為雲端原生生態的標準模式。越來越多的工具和平台以「Kubernetes 原生」的方式設計，意味著它們直接使用 Kubernetes API 進行擴展。

## 版本迭代

Kubernetes 在 2017 年發布了四個重要版本：

- **1.6**（3 月）：RBAC 授權、聯邦叢集
- **1.7**（6 月）：StatefulSet 更新、Pod 安全政策
- **1.8**（9 月）：自訂資源定義（CRD）beta、Volume 快照
- **1.9**（12 月）：Workload API 穩定、Windows Server 容器支援
