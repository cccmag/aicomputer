# 科技依存推論

**資訊科技年鑑　1950-2025**

編輯：[陳鍾誠](https://csie.nqu.edu.tw/p/404-1038-2377.php?Lang=zh-tw)

---

科技發展從來不是孤立事件，而是一條條環環相扣的長鏈。每一項新技術都建立在既有技術之上——沒有前人的基礎，就沒有後續的突破。這份文件從年鑑的 76 年跨度出發，梳理出資訊科技發展的關鍵依存關係。

## 1. 從圖靈機到人工智慧

```
圖靈機概念 (1936)
  └→ 圖靈測試 (1950)
  │    └→ 人工智慧 (1956)
  │         ├→ 專家系統 (1965)
  │         ├→ 機器學習 (1980s)
  │         │    └→ 深度學習 (2012)
  │         │         ├→ GPT-3 (2020)
  │         │         │    └→ 大型語言模型 (2020s)
  │         │         │         └→ AI 代理 (2025)
  │         │         └→ 電腦視覺 → 自動駕駛
  │         └→ 深度學習 → AlphaFold (2021)
  │              └→ 藥物設計革命
  └→ 儲存程式架構 (1945)
       ├→ ENIAC (1946) → SEAC/SWAC (1950)
       │    └→ 商用電腦 UNIVAC (1951)
       │         └→ 大型主機時代 (1960s)
       └→ 馮紐曼架構
            └→ 通用電子電腦
                 └→ 現代計算機體系
```

**推論**：沒有 1936 年圖靈的通用機概念，就沒有現代電腦的理論基礎。沒有 1950 年圖靈提出的「機器能否思考」問題，就沒有 1956 年達特矛斯會議開創的人工智慧領域。圖靈一個人的兩項貢獻——通用計算理論與 AI 願景——分別開啟了計算機科學與人工智慧兩大領域，這樣的影響力在科技史上極為罕見。

## 2. 從電晶體到摩爾定律

```
電晶體 (1947)
  └→ 積體電路 (1958)
       └→ 摩爾定律 (1965)
            ├→ 指數級成長
            │    ├→ Intel 4004 (1971)
            │    │    └→ 微處理器時代
            │    │         ├→ Intel 8080 (1974)
            │    │         │    └→ Altair 8800 (1975)
            │    │         │         └→ 個人電腦革命
            │    │         ├→ Intel 80386 (1985)
            │    │         │    └→ 32 位元運算
            │    │         │         └→ Windows 崛起
            │    │         └→ ARM 架構 (1985)
            │    │              └→ Apple M1 (2020)
            │    │                   └→ Apple M5 (2025)
            │    └→ 記憶體容量翻倍
            │         └→ 水銀延遲線 → 磁芯 → DRAM → SSD
            └→ 自我實現預言
                 └→ 半導體產業路線圖
```

**推論**：沒有 1947 年貝爾實驗室的電晶體，就沒有 1958 年的積體電路。沒有積體電路，就沒有摩爾定律驅動的指數成長。沒有這條鏈條，就沒有今日的千億電晶體晶片——你的手機運算能力將倒退到 1970 年代的大型主機水準。

## 3. 從大型主機到個人電腦

```
大型主機 (1950s)
  ├→ 迷你電腦
  │    ├→ PDP-8 (1965)
  │    │    └→ 迷你電腦普及
  │    │         └→ Homebrew Computer Club (1975)
  │    │              ├→ Apple II (1977)
  │    │              │    └→ 個人電腦家庭化
  │    │              └→ Microsoft Altair BASIC (1975)
  │    │                   └→ MS-DOS (1981)
  │    │                        └→ IBM PC (1981)
  │    │                             ├→ 相容機生態系
  │    │                             └→ Windows (1985)
  │    │                                  └→ Microsoft 帝國
  │    └→ CDC 6600 (1965)
  │         └→ Cray-1 (1976)
  │              └→ 超級電腦時代
  └→ 分時系統
       └→ Project MAC (1965)
            └→ Multics
                 └→ Unix (1969)
                      ├→ C 語言
                      │    └→ C++ (1985)
                      │         └→ 物件導向程式設計
                      └→ 開源軟體運動
                           └→ Linux (1991)
```

**推論**：沒有 PDP-8 開創的迷你電腦市場，就沒有 1975 年 Altair 8800 的誕生。沒有 Homebrew Computer Club 的業餘愛好者社群，就沒有 Apple 和 Microsoft。微電腦革命不是突然發生的——它是一條從大型主機到迷你電腦、再到微電腦的漸進路徑，每一步都降低了運算的門檻。

## 4. 從 ARPANET 到網際網路

```
ARPANET (1969)
  ├→ 乙太網路 (1973)
  │    └→ 區域網路標準化
  │         └→ DIX Ethernet (1980)
  │              └→ 辦公室網路化
  ├→ TCP/IP (1983)
  │    └→ NSFNET (1985)
  │         └→ 商用網際網路 (1990s)
  │              ├→ .com 域名 (1985)
  │              ├→ 全球資訊網 (1989)
  │              │    └→ Netscape Navigator (1994)
  │              │         └→ 瀏覽器大戰
  │              ├→ 搜尋引擎
  │              │    ├→ Archie (1990)
  │              │    ├→ Yahoo (1994)
  │              │    └→ Google (1998)
  │              │         └→ 搜尋廣告商業模式
  │              ├→ 達康泡沫 (2000)
  │              │    └→ Web 2.0 (2004)
  │              │         ├→ 社群媒體
  │              │         │    ├→ Facebook (2004)
  │              │         │    └→ Instagram (2010)
  │              │         └→ 雲端運算 (2006)
  │              │              ├→ Amazon Web Services (2006)
  │              │              └→ Microsoft Azure (2010)
  │              └→ 網際網路服務提供者
  │                   └→ 寬頻網路
  │                        └→ 線上影音串流
  └→ 電子郵件 (1971)
       └→ 網路通訊文化
```

**推論**：沒有 1969 年的 ARPANET，就沒有 TCP/IP 協定。沒有全球資訊網，就沒有 Netscape、Google 與 Facebook。網路效應（Metcalfe's Law）使得連線節點數量的成長帶來了價值的指數級增長——這就是網路從軍事研究計畫演變為人類文明最大平台的動力。

## 5. 從桌面到雲端

```
個人電腦普及 (1980s)
  ├→ 圖形使用者介面
  │    ├→ Macintosh (1984)
  │    │    └→ Mac OS X (2001)
  │    │         └→ iOS (2007)
  │    │              ├→ iPad (2010)
  │    │              │    └→ 後 PC 時代
  │    │              └→ App Store 生態系
  │    │                   └→ Swift 程式語言 (2014)
  │    └→ Windows 3.0 (1990)
  │         └→ Windows 95/98/2000
  │              └→ PC 桌面壟斷
  ├→ 套裝軟體
  │    ├→ Microsoft Office
  │    └→ Photoshop (1990)
  │         └→ 數位影像處理產業
  └→ 前端開發技術
       ├→ JavaScript
       │    └→ AngularJS (2010)
       │         └→ Angular (2016)
       │    └→ React (2013)
       │    └→ Vue.js (2014)
       └→ 動態網頁 → Web 應用程式
            └→ Web 2.0 互動體驗
```

**推論**：個人電腦的普及創造了軟體產業。沒有 GUI，電腦就不會從專業工具變成大眾消費品。沒有圖形介面，就沒有 iPhone 的多點觸控——觸控是 GUI 理念在更小螢幕上的極致延伸。Mac OS X 的 Unix 核心、Aqua 介面和觸控手勢，直接孕育了 iOS 的誕生。

## 6. 從數據機到物聯網

```
數據機 (1950s)
  ├→ 撥接上網 (1990s)
  │    └→ 寬頻網路 (2000s)
  │         └→ 永遠在線的網路
  │              ├→ 串流媒體
  │              │    ├→ YouTube (2005)
  │              │    └→ Netflix 串流 (2007)
  │              ├→ Skype (2003)
  │              │    └→ 網路電話 → Zoom (2011)
  │              │         └→ 遠距工作 → COVID-19 疫情 (2020)
  │              └→ 雲端遊戲
  │                   └→ PlayStation 2 (2000)
  │                        └→ Xbox Live (2002)
  └→ 無線通訊
       ├→ 藍牙 (2000)
       │    └→ 藍牙 4.0 BLE (2010)
       │         └→ 物聯網爆炸
       │              └→ 智慧家庭
       └→ Wi-Fi (1997)
            └→ 行動網路 → 3G/4G/5G
                 └→ 智慧手機生態系
```

**推論**：沒有數據機就沒有撥接網路，沒有寬頻就沒有永遠在線的連線，沒有永遠在線就沒有串流媒體、雲端服務與遠距工作。從 56K 撥接聲到 Zoom 上的百萬人會議，網路頻寬的百萬倍成長改變了工作、娛樂與社交的型態。

## 7. 從達康泡沫到雲端運算

```
達康泡沫破滅 (2000-2002)
  ├→ 基礎設施未被摧毀
  │    └→ 光纖骨幹、伺服器農場、寬頻覆蓋
  │         └→ 下一波創新的基礎
  ├→ 倖存者效應
  │    ├→ Google (搜尋廣告)
  │    │    └→ 張量處理單元 (2016)
  │    │         └→ AI 訓練基礎設施
  │    ├→ Amazon (電子商務基礎設施)
  │    │    └→ Amazon Web Services (2006)
  │    │         └→ 雲端運算時代
  │    │              ├→ Microsoft Azure (2010)
  │    │              └→ 企業 IT 轉型
  │    │                   └→ AI 服務雲端化
  │    │                        └→ GPT-3 API (2020)
  │    └→ eBay (市集模式)
  └→ Web 2.0 (2004)
       └→ 使用者生成內容
            ├→ 社群媒體 (Facebook, Instagram)
            └→ SaaS 商業模式
```

**推論**：達康泡沫雖然消滅了無數「眼球經濟」新創，卻留下了光纖網路、伺服器農場和寬頻覆蓋率。沒有這些基礎設施，就沒有雲端運算。沒有雲端運算的彈性算力，就沒有大規模 AI 訓練。泡沫的毀滅性清理，恰好為下一波技術革命準備了土壤。

## 8. 從量子力學到量子運算

```
量子力學 (1925-1927)
  ├→ 能帶理論 (1928)
  │    └→ 半導體物理
  │         └→ 電晶體 (1947)
  │              └→ 摩爾定律 → 物理極限
  │                   └→ 量子效應成為限制
  │                        └→ 量子霸權追求 (2019)
  │                             └→ Google Sycamore (2019)
  │                             └→ Google Willow (2025)
  │                                  └→ 可驗證量子優勢
  └→ 量子位元概念
       └→ Shor 演算法 (1994)
            └→ 量子密碼學
            └→ 量子運算硬體
                 ├→ 超導量子位元
                 │    └→ Google / IBM 路線
                 └→ 離子阱
                      └→ IonQ / Honeywell 路線
```

**推論**：沒有量子力學就沒有半導體物理，沒有半導體就沒有現代計算機。而當電晶體縮小到原子尺度時，量子效應從「工程問題」變成了「物理限制」——這迫使我們反向利用量子力學來進行計算。摩爾定律的終點，恰好是量子運算的起點。

## 9. 從儲存程式到 AI 代理

```
儲存程式架構 (1945)
  └→ 通用計算機
       └→ 軟體概念誕生
            ├→ 作業系統
            │    ├→ 分時系統 (1960s)
            │    └→ 虛擬化 (1970s)
            │         └→ 雲端運算 (2006)
            ├→ 程式語言演化
            │    ├→ BASIC (1965) → 大眾程式設計
            │    ├→ C (1972) → Unix → 系統軟體
            │    └→ C++ (1985) → 物件導向
            └→ AI 軟體
                 ├→ 專家系統 (1965)
                 ├→ 機器學習 (1980s)
                 │    └→ 深度學習 (2012)
                 │         └→ GPT-3 → ChatGPT (2022)
                 │              └→ 大型語言模型
                 │                   └→ 多模態 AI (2023-2025)
                 │                        └→ AI 代理 (2025)
                 │                             └→ Claude Code
                 │                             └→ MCP 協定
                 └→ 張量處理單元 (2016)
                      └→ AI 專用硬體
                           └→ Nvidia Blackwell (2025)
```

**推論**：儲存程式架構不僅是硬體設計，更開創了「軟體」的概念——將程式與硬體分離。這個分離使得技術演進可以分軌進行：硬體遵循摩爾定律，軟體則在抽象層次上不斷攀升。從機器語言到高階語言，從函式庫到 API，從應用程式到 AI 代理——每一次抽象層次的提升，都釋放了新的生產力。MCP 協定（2025）讓 AI 代理可以像人類使用工具一樣呼叫服務，這是抽象化趨勢的最新篇章。

## 10. 硬體的歷史輪迴：從大型主機到 AI 晶片

```
大型主機 (1950s-1960s)
  └→ 集中運算、共享資源、專業管理

       ↓ 典範轉移
個人電腦 (1975-2000)
  └→ 分散運算、個人擁有、使用者管理

       ↓ 典範轉移
雲端運算 (2006-)
  └→ 集中運算、共享資源、專業管理
       └→ 大型資料中心 → AI 訓練叢集

       ↓ 典範轉移
邊緣運算 + AI 晶片 (2020s-)
  └→ 分散運算、個人擁有、AI 加速
       └→ Apple M5 Neural Engine
       └→ Nvidia Blackwell GPU
```

**推論**：科技發展常以螺旋形式重複自己。從大型主機的集中運算，到 PC 的分散運算，再到雲端的集中運算——運算型態在「集中」與「分散」之間擺盪。每一次擺盪都不是簡單的重複，而是在更高層次上的回歸。2025 年，我們正處於 AI 專用硬體驅動的新一輪分散化進程中。

## 12. 結論：技術樹與路徑依賴

從這份年鑑的 76 年跨度來看，資訊科技的發展呈現鮮明的木本植物式（樹狀）結構：

**每一項重大技術都有一個不可跳過的先決條件節點：**
- 沒有 **圖靈機概念** → 就沒有儲存程式架構 → 也就沒有現代電腦
- 沒有 **電晶體** → 就沒有積體電路 → 也就沒有摩爾定律驅動的指數成長
- 沒有 **ARPANET** → 就沒有 TCP/IP → 也就沒有網際網路
- 沒有 **全球資訊網** → 就沒有搜尋引擎 → 也就沒有 Google 與現代網路經濟
- 沒有 **深度學習** → 就沒有大型語言模型 → 也就沒有 AI 代理革命

**先決條件的先決條件：**
- 沒有 **布林代數** (1854) → 就沒有邏輯電路 → 就沒有現代計算機
- 沒有 **電磁學** (1864) → 就沒有無線電 → 就沒有乙太網路與 Wi-Fi
- 沒有 **量子力學** (1925) → 就沒有半導體 → 就沒有整個資訊時代
- 沒有 **數學** → 什麼都沒有

**路徑依賴**：一旦選擇了某條技術路線，後續發展就被鎖定在這條路徑上。例如：
- x86 架構 → 數十年的軟體相容性累積 → 即使 ARM 更高效也難以撼動桌面市場
- QWERTY 鍵盤 → 即使效率不是最佳，也無法改變
- 矽基半導體 → 整個產業鏈圍繞矽建立 → 碳化矽等新材料需數十年滲透
- Windows API → 數十萬應用程式的生態系 → Microsoft 數十年的市場主導
- TCP/IP → 整個網際網路的基礎協定 → 即使有更好的協定也無法取代

**創新並非憑空出現，而是在既有技術的邊界上向前邁出的一小步**。了解這些依存關係，不僅幫助我們理解歷史，更能指引未來的創新方向——真正的突破往往來自於找到並解開那些制約下一步發展的關鍵節點。
