# BASIC 語言的擴散：從達特茅斯走向世界

## 為非科學家設計的語言

1964 年，約翰·凱梅尼（John Kemeny）和湯瑪斯·庫爾茨（Thomas Kurtz）在達特茅斯學院創造了 BASIC 語言。到了 1965 年，BASIC 開始從達特茅斯向外擴散，吸引了越來越多教育機構和電腦使用者的注意。BASIC 的設計目標非常明確：讓非科學專業的學生也能夠輕鬆學習程式設計。

1960 年代初期，學習程式設計的唯一方式是掌握 FORTRAN 或 ALGOL 等科學計算語言，這些語言的語法對初學者並不友善。凱梅尼和庫爾茨觀察到達特茅斯的學生——尤其是人文和社會科學領域的學生——對程式設計有著濃厚的興趣，卻被 FORTRAN 的複雜性擋在門外。

BASIC 的設計原則可以歸納為三個詞：簡單、互動、親切。它的語法使用英文單詞如 LET、PRINT、INPUT、IF-THEN，使得完全沒有數學背景的使用者也能理解程式在做什麼。錯誤訊息以平易近人的方式呈現，而非令人困惑的錯誤代碼。

## 達特茅斯分時系統的催化作用

BASIC 的成功與達特茅斯分時系統（DTSS）密不可分。1965 年，DTSS 已經穩定運作，讓學生可以使用電傳打字機終端機來編寫和執行 BASIC 程式。這種即時的互動體驗——鍵入一行程式碼，按下 Enter，馬上看到結果——與傳統的打卡批次處理模式截然不同。

在當時，大多數學校的學生必須將程式碼寫在打孔卡片上，交給計算機中心排隊執行，幾個小時甚至一天後才能拿到結果。BASIC 搭配 DTSS 的組合，讓程式設計變成了與電腦即時對話的過程，這大大降低了學習的挫折感和門檻。

## 教育領域的迅速普及

1965 年，BASIC 開始在美國東北部的大學中傳播。麻省理工學院的一些研究團隊對 BASIC 表現出濃厚興趣，因為他們發現這個語言可以讓實驗室的研究人員（而非專業程式設計師）快速地開發小型應用。

到 1965 年底，已有十幾所大學安裝了 BASIC 系統。凱梅尼和庫爾茨慷慨地將 BASIC 的實作細節公開發表，並鼓勵其他機構自行移植。這種開放的態度與當時 IBM 對其軟體的封閉態度形成鮮明對比。

## 對未來的深遠貢獻

BASIC 對電腦普及的貢獻怎麼強調都不為過。在 1970 年代，微軟的比爾·蓋茲和保羅·艾倫為 Altair 8800 寫了第一個微電腦版本的 BASIC，這成為微軟公司的第一個產品。在 1980 年代，幾乎所有的個人電腦——包括 Apple II、Commodore 64 和 IBM PC——都內建了 BASIC 直譯器。

BASIC 的「低地板、高天花板」設計——容易入門但功能完整——影響了後來幾乎所有的教育用程式語言和環境，包括 Python、Scratch 和 JavaScript。當我們今天在瀏覽器中鍵入 `console.log()` 並即時看到結果時，我們正在體驗 1965 年達特茅斯學生所感受到的那種即時回饋的喜悅。
