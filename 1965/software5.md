# Project MAC 與早期分時概念

## MAC 的雙重意義

1963 年成立於 MIT 的 Project MAC 在 1965 年已經成為全球分時系統研究的中心。MAC 這個縮寫有雙重意義：「Machine-Aided Cognition」（機器輔助認知）和「Multiple-Access Computer」（多重存取電腦），這兩種解釋精確地捕捉了這個計畫的兩個核心目標：用電腦擴展人類智慧，以及讓多人同時使用同一部電腦。

Project MAC 的核心硬體是一台經過大幅改造的 IBM 7094，配備了 32,768 個 36 位元字組的磁芯記憶體（約 144 KB）。這台機器被命名為「IBM 7094 Modified」或簡稱「7094M」。為了支援分時運作，MIT 的工程師對硬體進行了全面改造，包括增加了記憶體保護機制、中斷控制器、以及多組暫存器組（register set）以加速使用者切換。

## CTSS 的運作經驗

到 1965 年，CTSS 已經在 Project MAC 運行了將近兩年，積累了大量的分時系統運作經驗。CTSS 支援多達 30 個同時連線的使用者，每人透過一台 Teletype Model 33 或 IBM 1050 終端機進行互動。每個使用者每分鐘獲得約 0.5 秒的 CPU 時間切片——對於互動式編輯和除錯來說已經足夠。

CTSS 的使用者經驗讓研究人員清楚地看到了分時系統的優勢和挑戰。優勢是顯而易見的：使用者不再需要等待數小時的批次處理排隊，可以在幾秒鐘內得到回應。挑戰則包括了排程演算法的設計、記憶體保護的實現、以及如何防止一個使用者的錯誤程式影響到其他使用者。

## 分時排程技術的演進

Project MAC 的團隊在 1965 年對分時排程進行了重要的研究。他們發現，簡單的輪詢（round-robin）排程在互動負載下表現良好，但當某些使用者執行計算密集任務時，整體回應時間會急遽惡化。

為了解決這個問題，Corbato 和團隊開發了多層級回饋佇列排程演算法。這個演算法的基本概念是：短時間的互動任務獲得高優先級，長時間的計算任務則逐漸降低優先級。這個排程策略後來成為所有現代作業系統排程器的基礎。

另一個重要的貢獻是「自願交出 CPU」（voluntary relinquishment）的概念。CTSS 允許正在等待 I/O 完成的程式主動交出 CPU 時間，讓排程器可以將資源分配給其他需要 CPU 的程式。這種設計在後來的 Unix 和 Linux 中繼續沿用。

## 對未來的影響

Project MAC 在 1965 年的研究成果直接影響了後續的作業系統設計。Multics（1965 年啟動）的虛擬記憶體和保護機制設計大量借鑑了 CTSS 的經驗。Unix（1969）在設計上雖然刻意與 Multics 的複雜性保持距離，但其行程管理和檔案系統的核心概念仍然源自 CTSS 和 Project MAC 的研究成果。

Project MAC 的模式——一個大學實驗室同時進行系統研究、軟體開發和大規模使用者服務——後來成為許多電腦科學研究機構的典範。伯克利的 BSD Unix 專案、卡內基美隆的 Mach 微核心專案、以及 MIT 自己的 X Window 系統，都沿襲了 Project MAC 的研究模式。
