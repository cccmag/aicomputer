# Project MAC：MIT 大規模分時系統

## 分時系統的旗艦

1965 年，MIT 的 Project MAC 已成為全世界最大、最具影響力的分時系統研究計畫。這項由 ARPA 資助的計畫，以一台高度改造的 IBM 7094 電腦為核心，支援多達 30 位使用者同時透過遠端終端機進行互動操作。在一個電腦仍然以批次處理為主的時代，Project MAC 展示了一種全新的使用電腦的方式。

Project MAC 的技術基礎建立在 CTSS（相容分時系統）之上。CTSS 是第一個成功運作的分時系統，由 MIT 的 Fernando Corbato 帶領團隊在 1963 年完成。到了 1965 年，CTSS 已經從一個研究原型進化為一個穩定可靠的大型系統，每天服務來自全 MIT 校園的研究人員和學生。

## 使用者社群的形成

Project MAC 最引人注目的成就之一，是創造了第一個大型的電腦使用者社群。研究人員不再獨自在機房中操作電腦，而是透過終端機從遠端連接到一個共享的計算資源池。這種模式改變了人們與電腦的關係——電腦不再是需要預約的稀缺資源，而是隨時可用的工具。

這個社群的成員來自 MIT 的各個系所，包括數學、物理、電機工程、語言學、和經濟學等領域。許多使用者的背景並非電腦科學，而是他們各自領域的研究人員。這使得 Project MAC 成為了跨學科合作的典範，並展示了計算工具如何能夠促進各種領域的研究。

## 終端機與使用者介面

Project MAC 的使用者主要透過 Teletype Model 33 ASR 和 IBM 1050 終端機與系統互動。這些終端機透過低速的電話線路連接到 IBM 7094 上的福特介面（Ford Interface，因為由福特汽車公司捐贈）。終端機列印的速度約為每秒 10 到 15 個字元，在當時已經被視為快速。

系統提供了一系列工具來支援使用者的日常工作。編輯器（主要是 TECO 和後來的 QED）允許使用者建立和修改文字檔案。檔案系統支援目錄結構，每個使用者擁有自己的檔案目錄。系統管理者可以設定使用者權限和資源限制，以確保沒有人能夠獨佔系統資源。

## 研究與服務的雙重使命

Project MAC 同時肩負研究和服務的雙重使命。一方面，它是一個前沿的電腦科學研究實驗室，研究人員在這裡開發新的作業系統概念、程式語言和互動技術。另一方面，它也是一個提供日常計算服務的公用設施，為 MIT 校園內的研究和教育活動提供支援。

這種雙重使命有時會產生衝突——研究需要的實驗變更可能會影響服務的穩定性——但更多的是相互促進。研究人員將新的想法部署在系統上接受真實使用者的考驗，而使用者需求的回饋也幫助研究人員調整他們的研究方向。

Project MAC 的模式——將一個研究實驗室與一個大型共用計算系統結合——直接影響了後來的 Xerox PARC（1970）和微軟研究院（1991）的研究模式。而分時系統創造的「共用計算資源、多人異地存取」概念，更是當今雲端運算和軟體即服務（SaaS）模式在 1965 年的先行者。
