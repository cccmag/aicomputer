# 歷史穿梭：1965 年的種子與未來的大樹

## 從 PDP-8 到個人電腦

1965 年 PDP-8 以 18,000 美元的價格突破了電腦的價格障礙，證明了低價電腦的商業可行性。這條路線的演化清晰可見：

- **1965**：PDP-8（12 位元、18,000 美元）
- **1970**：PDP-11（16 位元、10,000 美元起）
- **1975**：Altair 8800（Intel 8080、397 美元套件）
- **1977**：Apple II（完整個人電腦、1,298 美元）
- **1981**：IBM PC（個人電腦的標準化）

Gordon Bell 的 PDP-8 設計哲學——保持簡潔、降低價格——被 Edson de Castro 帶到了 Data General 的 Nova（1969），然後被 Chuck Peddle 帶到了 MOS Technology 的 6502 處理器（1975），最終出現在 Apple II 和 Commodore 64 中。當 Steve Jobs 和 Steve Wozniak 在車庫中組裝 Apple I 時，他們正是 PDP-8 開創的平價電腦傳統的繼承者。

## 從 CDC 6600 到 Cray-1 再到 GPU 運算

Seymour Cray 的 CDC 6600 以 3 MFLOPs 的效能開啟了超級電腦的時代。Cray 在 CDC 6600 中展示的設計理念——以極致的工程品質追求最高效能——成為超級電腦設計的黃金標準：

- **1965**：CDC 6600（3 MFLOPs，10 個 PPU）
- **1969**：CDC 7600（10 MFLOPs）
- **1976**：Cray-1（160 MFLOPs，向量處理器）
- **1985**：Cray-2（1.9 GFLOPs，液體冷卻）

有趣的是，CDC 6600 的「多個專用處理器輔助主處理器」架構，在半個世紀後以全新的形式再現——GPU（圖形處理器）的 CUDA 架構正是使用數千個小型核心來進行平行運算，與 CDC 6600 的周邊處理器理念有著驚人的相似之處。

## 從 Multics 到 Unix 到 Linux

1965 年啟動的 Multics 專案雖然未能達成所有原定目標，卻透過否定之否定塑造了現代作業系統的面貌：

- **1963**：CTSS（MIT，第一個分時系統）
- **1965**：Multics 啟動（MIT/Bell Labs/GE）
- **1969**：貝爾實驗室退出 Multics
- **1969-71**：Unix 誕生（Thompson、Ritchie）
- **1973**：用 C 語言重寫 Unix
- **1984**：Richard Stallman 啟動 GNU 計畫
- **1991**：Linus Torvalds 發布 Linux

Thompson 對 Multics 的回應是：保持小而美。他的 Unix 將 Multics 複雜的概念簡化為優雅的基本元素——檔案作為位元組串流、管道作為行程間通信、模組化的工具。而這一切，始於 1965 年 MIT 校園內那一場關於「如何讓電腦為多人服務」的討論。

## 摩爾定律：半世紀的自我實現預言

戈登·摩爾在 1965 年提出的觀察，不僅是預測，更成為驅動力量。半導體公司以「跟上摩爾定律」為目標投入研發，使得這個預言自我實現：

- **1971**：Intel 4004（2,300 個電晶體）
- **1978**：Intel 8086（29,000 個電晶體）
- **1993**：Intel Pentium（310 萬個電晶體）
- **2008**：Intel Core i7（7.31 億個電晶體）
- **2022**：Apple M2 Ultra（1,340 億個電晶體）

從 1965 年到 2020 年，摩爾定律持續了超過 55 年，將電晶體的密度提升了超過 8 個數量級。這段期間內，一部智慧型手機的運算能力超越了 1965 年地球上所有電腦的總和。

## 從 BASIC 到全民寫程式

BASIC 語言在 1964 年的誕生和 1965 年的擴散，寫下了「讓每個人都能寫程式」這個理念的起點：

- **1964-65**：BASIC 在達特茅斯誕生和擴散
- **1975**：Altair BASIC（比爾·蓋茲和保羅·艾倫的首個產品）
- **1977**：Apple II 內建 BASIC
- **1981**：IBM PC 內建 BASICA
- **1991**：Python 誕生（強調可讀性和易學性）
- **2007**：MIT 推出 Scratch（圖形化程式設計教育）
- **2010 年代**：Code.org 等全民程式設計教育運動

每一代「全民程式語言」——從 BASIC 到 Python 再到 Scratch——都在降低程式設計的門檻，繼承著 1965 年凱梅尼和庫爾茨的初衷。

## 分時到雲端

Project MAC 在 1965 年展示的「多人同時使用同一台電腦」的概念，最終演化為雲端運算這個 21 世紀最重要的運算模式：

- **1965**：CTSS 在 Project MAC（30 人共用 IBM 7094）
- **1970s**：分時服務商業化（GE、Tymshare）
- **1990s**：應用服務供應商（ASP）
- **2006**：Amazon Web Services（EC2、S3）
- **2010s**：雲端運算全面普及

當 2025 年的一位開發者使用 AWS Lambda 執行無伺服器函數時，他在做的事與 1965 年 Project MAC 使用者在 Teletype 終端機上執行命令本質上是相同的——只是規模擴大了數百萬倍。

## 1965 的時代輪廓

1965 年除了科技的進展，還是一個充滿變動的時代：美國在越南戰場逐步升級，民權運動如火如荼，Beatles 發行了《Rubber Soul》專輯，Bob Dylan 在紐波特民謠音樂節上「插電」演出。在這個傳統與革命碰撞的年代，資訊科技的突破並非孤立事件，而是人類社會對「自動化」、「效率」、「連結」等需求的自然回應。

1965 年的每一項突破——從 PDP-8 到摩爾定律到 Multics——都像投入時間長河的種子，在往後的半個世紀中，長成了改變全人類生活的參天大樹。
