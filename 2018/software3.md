# GitHub 被 Microsoft 收購

## 震驚開源社群的消息

2018 年 6 月 4 日，Microsoft 宣布以 75 億美元（全股票交易）收購 GitHub——全球最大的開源程式碼託管平台。這個消息在開源社群引發了激烈討論：畢竟 Microsoft 過去長期被視為開源軟體的反對者（Steve Ballmer 曾在 2001 年稱 Linux 為「癌症」）。

GitHub 成立於 2008 年，到 2018 年已經擁有超過 2800 萬開發者和 8500 萬個倉庫。Microsoft 本身也是 GitHub 上最大的貢獻者——超過 1000 名 Microsoft 員工在 GitHub 上積極參與開源專案。

## 承諾與變革

新任 Microsoft CEO Satya Nadella 在收購公告中強調了三個承諾：

1. GitHub 將保持平台獨立性，繼續支援所有語言、框架和工具
2. GitHub 的營運將繼續由原團隊負責
3. Microsoft 的文化正在改變，開源是核心策略

收購後，GitHub 推出了多項重要變革：

- **免費私有倉庫**：2019 年 1 月開始提供不限量的免費私有倉庫（最多 3 個協作者）
- **包管理**：GitHub Package Registry
- **GitHub Actions**：內建 CI/CD 功能
- **Sponsors**：贊助開源維護者

## 收購的意義

這次收購是微軟轉型策略的關鍵一步。「開發者優先」是 Satya Nadella 自 2014 年上任以來的核心方針。收購 GitHub 讓 Microsoft 直接掌握了開發者社群的核心平台——這在雲端運算（Azure vs AWS vs GCP）的競爭中至關重要。

後來的發展證明：GitHub 在 Microsoft 旗下確實保持了相對獨立的營運，活躍用戶從 2800 萬成長到超過 1 億（2022 年）。
