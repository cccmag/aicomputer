# Windows 8 與 Surface RT 的觸控方向

## 微軟的觸控賭注

2012 年 10 月 26 日，Microsoft 正式發表 Windows 8。這不是一次普通的 Windows 升級——它是從核心到介面的全面改造。Windows 8 引入了 Metro（後稱 Modern UI）設計語言，以動態磚（Live Tiles）取代了傳統的開始功能表。

Windows 8 的設計假設是：未來的 PC 將是觸控螢幕的，傳統的鍵盤滑鼠互動將不再是主流。這個假設來自 iPad 和 iPhone 對消費電子市場的衝擊。Microsoft 試圖用一個作業系統同時服務觸控平板和傳統桌面兩種模式。

## 開始功能表的消失

Windows 8 最具爭議的改變是取消了開始功能表。取而代之的是開始畫面——一個全螢幕的動態磚面板。使用者不再能看到桌面、執行中的程式、系統托盤。對於習慣了 Windows 95 以來操作方式的企業用戶，這個改變難以接受。

Windows 8 也引入了 Windows Store 應用程式商店，採用新的 WinRT API（而非傳統的 Win32）。這套新架構基於類似 iOS 的沙盒安全模型，但缺少對既有 Win32 應用程式的支援。

## Surface RT 硬體

同一天，Microsoft 推出了 Surface RT——第一款由 Microsoft 自行設計和生產的個人電腦。Surface RT 搭載 ARM 架構的 Nvidia Tegra 3 處理器，10.6 吋螢幕，運行 Windows RT——一個無法執行傳統 x86 應用程式的 Windows 8 版本。

Surface RT 的市場表現慘淡。消費者不理解為何它不能執行既有的 Windows 軟體，續航和效能也不如 iPad。Microsoft 最終在 2013 年對 Surface RT 庫存提列了 9 億美元的減值損失。
