# 歷史穿梭：1974 年的種子

## 從 TCP 到網際網路

1974 年 Cerf 與 Kahn 的 TCP 論文，是網際網路歷史最重要的單一文件之一：

**NCP (1970)** → **TCP 論文 (1974)** → **TCP/IP 分家 (1978)** → **ARPANET 改用 TCP/IP (1983)** → **全球網際網路**

論文中提出的閘道器概念和資料報模型，為不同網路之間的互連提供了理論基礎。有趣的是，Cerf 和 Kahn 在 1973 年創造了「internet」這個詞（inter-networking 的縮寫），但這個詞要到 1980 年代才普及。

## CP/M 與軟體生態系的先驅

Gary Kildall 的 CP/M 在 1974 年只是一套在 Intel 開發系統上運行的磁碟作業系統，但其影響深遠：

**CP/M (1974)** → **大量應用軟體** → **MS-DOS 的模仿 (1981)** → **Windows 的崛起**

CP/M 引入的 BIOS 概念——將硬體相依代碼隔離在一個可替換的模組中——是軟體工程的一大創新。這種架構讓作業系統可以移植到不同的硬體平台上，後來被 MS-DOS 和 Linux 所繼承。

## Intel 8080 的架構傳承

8080 的指令集設計直接影響了 x86 架構，這是電腦史上最長的架構傳承：

**8008 (1972)** → **8080 (1974)** → **8086 (1978)** → **80286 - 80486** → **Pentium** → **Core i 系列**

Zilog 的 Federico Faggin（8080 的設計者之一）在離開 Intel 後創造了 Z80，與 8080 相容但更便宜。Z80 後來被用於 TRS-80、Commodore、Game Boy 等數不清的系統中。

## 從 Prolog 到邏輯程式設計

Prolog 雖然在 1974 年仍只是法國馬賽大學的一個研究專案，但它代表了一條與主流截然不同的程式設計路線：

**Q-Systems (1970)** → **Prolog (1972)** → **擴散 (1974-75)** → **第五代電腦** → **約束邏輯程式設計** → **Answer Set Programming**

Prolog 的邏輯程式設計思維在 AI 領域留下深遠影響，特別是專家系統和自然語言處理。雖然它從未成為主流商業語言，但其對宣告式程式設計的推動為後來的 SQL、規則引擎和知識表示系統奠定了基礎。

## 結構化程式設計的革命

1974 年，結構化程式設計從學術理論滲透到軟體開發實踐：

**goto 討論 (1968)** → **結構化理論 (1970s)** → **Pascal 的普及** → **C 語言的設計** → **物件導向**

結構化程式設計的核心理念—順序、選擇、迴圈—成為所有現代程式語言的基礎。雖然後來的物件導向、函數式程式設計補充了結構化方法的不足，但結構化程式設計的基本原則從未被取代。

## 1974 年的時代脈絡

1974 年的科技發展與時代背景緊密相連：

- **石油危機（1973-1974）**：全球石油危機導致經濟衰退，但也促進了能源效率和自動化的需求
- **Vietnam 戰爭尾聲**：DARPA 在軍事通訊網路上的投資直接催生了 TCP/IP 和 PRNET
- **反壟斷行動**：美國司法部對 IBM 的反壟斷訴訟（1969-1982）影響了 IBM 的市場策略，為新興的微電腦公司創造了空間
- **Intel 的成功轉型**：Intel 從記憶體製造商轉向微處理器公司，8080 的成功確立了其市場領導地位

1974 年最迷人處在於：TCP、CP/M、8080 這些當時看似獨立的事件，日後將交織成我們今天所知的數位世界。
