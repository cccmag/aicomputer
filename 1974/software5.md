# Pascal 語言的擴散與影響

## Wirth 的語言設計哲學

Niklaus Wirth 在 1971 年發表了 Pascal 語言，並在蘇黎世聯邦理工學院（ETH Zurich）實現了第一個編譯器。到了 1974 年，Pascal 正從教學工具轉變為實用程式語言。

Pascal 的設計以清晰、簡潔、結構化為核心原則。Wirth 的哲學是：程式語言的設計應該幫助程式設計師寫出正確的程式，而不是提供最多的功能。因此 Pascal 捨棄了 Fortran 的複雜 I/O、COBOL 的資料描述、ALGOL 66 的某些進階特性，提供了小而美的語言核心。

## 1974 年的關鍵發展

1974 年，Pascal 的使用者手冊（Pascal User Manual and Report）由 Wirth 和 Kathleen Jensen 合著出版，成為 Pascal 程式設計師的標準參考文件。同時，Pascal 編譯器開始移植到更多的機器平台：

- CDC 6000 系列（Wirth 原版）
- IBM System/370
- DEC PDP-11
- 各種迷你電腦

Pascal 的移植性成為它最大的優勢之一。不像 Fortran 和 BASIC 那樣各家廠商有各自的方言，Pascal 的標準相對統一。

## 對教育的影響

Pascal 在 1970 年代末到 1980 年代成為大學程式設計入門課程的首選語言。它強制程式設計師使用結構化控制結構、強調型別安全、要求變數宣告——這些在當時都是革命性的教育理念。

美國 AP（Advanced Placement）電腦科學課程在 1984 年選擇 Pascal 作為考試語言，進一步鞏固了其在教育領域的地位，直到 1999 年被 C++ 取代。
