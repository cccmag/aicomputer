# 歷史穿梭：1971 年的種子

## 微處理器的開枝散葉

1971 年 Intel 4004 的誕生，開啟了人類科技史上最輝煌的指數成長曲線：

**Intel 4004 (1971, 2,300 電晶體) → 8008 (1972, 3,500) → 8080 (1974, 6,000) → 8086 (1978, 29,000) → 80386 (1985, 275,000) → Pentium (1993, 3.1M) → Core i9 (2020s, 數十億)**

從計算器晶片到現代處理器，4004 的基本架構——ALU、控制單元、暫存器陣列——仍然是今天所有 CPU 的核心。差異只在於規模從 2,300 個電晶體成長到數百億個。

## 從公用計算到雲端運算

McCarthy 的公用計算概念經歷了迂迴的路徑：

**公用計算 (1961) → 分時系統 (1960s) → 個人電腦的分散化 (1980s) → 網際網路 (1990s) → Amazon Web Services (2006) → 現代雲端運算**

每次我們使用 Gmail、Google Drive 或 AWS 時，我們都是在兌現 McCarthy 在 1961 年提出的願景。他的洞見在 1971 年圖靈獎的肯定下，從學術界的邊緣想法成長為整個資訊產業的骨幹。

## 電子郵件如何改變世界

Tomlinson 在 1971 年發明的電子郵件是網路史上第一個「殺手級應用」：

**SNDMSG + CPYNET (1971) → email (1971) → ARPANET 流量爆炸 → 商業化電子郵件 (1980s) → Webmail (1990s) → 智慧型手機電子郵件 (2000s)**

電子郵件從一個簡單的側專案（Tomlinson 從未為它申請專利）成長為人類歷史上最廣泛使用的非同步通訊工具。它改變了商業溝通、社會關係和資訊傳播的方式。

## 結構化程式設計 vs 軟體危機

1971 年 Brooks 的《人月神話》描繪了大型軟體開發的困境，而結構化程式設計被視為解方：

**結構化定理 (1966) → Go To 爭論 (1968) → 逐步求精 (1971) → 物件導向 (1980s) → 極限編程 (1999) → 敏捷開發 (2001)**

軟體工程的發展史，從某種角度來看，就是人類對抗軟體複雜度的永恆戰爭。

## 從 troff 到數位排版

**troff (1971) → TeX (1978) → PostScript (1984) → PDF (1993) → 網頁排版 (CSS, 1996)**

troff 的開發動機——讓文字排版自動化——推動了一個持續數十年的數位排版革命。今天我們在網頁和電子書中看到的排版品質，其根源可以追溯到 1971 年貝爾實驗室專利部門那個簡單的排版工具。

## 1971 年的時代脈絡

1971 年的技術發展反映了更廣泛的社會變遷：
- **越戰的尾聲**：ARPANET 源自國防經費，但到 1971 年已經開始轉向學術和民用
- **美國的經濟轉型**：大型主機市場飽和，小型化和個人化計算的種子正在發芽
- **太空探索的延伸**：4004 被安裝在先鋒 10 號上（1972 年發射），成為第一個飛出太陽系的人造物體的控制中樞
- **知識工作的自動化**：電子郵件和文書處理工具正在改變辦公室工作的本質

1971 年最令人驚嘆的是：在這一年結出的種子——微處理器、電子郵件、公用計算、結構化軟體——各自獨立發展，數十年後卻在個人電腦和網際網路的世界中完美地交匯在一起。
