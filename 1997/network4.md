# Internet Explorer 4.0 與桌面整合

## 瀏覽器與作業系統的融合

1997 年 9 月發布的 Internet Explorer 4.0 不僅僅是一個瀏覽器更新——它代表了 Microsoft 將網際網路整合到 Windows 作業系統中的願景。這個策略被稱為「Active Platform」，核心是讓網際網路成為作業系統的自然延伸。

IE 4.0 的安裝過程本身就會修改 Windows 的桌面外觀和行為。使用者可以選擇「Windows 桌面更新」——這會將瀏覽器的導航按鈕（上一頁、下一頁）加入到檔案總管中。

## Active Desktop

Active Desktop 是 IE 4.0 最具話題性的功能。它將 HTML 渲染引擎嵌入到 Windows 桌面上——使用者可以：

- 將網頁設為桌面背景
- 在桌面上放置即時更新的小工具（股票行情、天氣、新聞）
- 桌面圖示變成活動的 HTML 元素

Active Desktop 在技術上令人印象深刻，但在實用上存在問題：它佔用大量系統資源，而且在 1997 年的電腦上執行非常緩慢。

## 對瀏覽器戰爭的戰略意義

IE 4.0 的桌面整合策略從幾個方面幫助了 Microsoft 的瀏覽器戰爭：

- 當使用者安裝 IE 4.0 時，作業系統的體驗也隨之改變——這增加了使用者切換到 Netscape 的心理成本
- IE 4.0 的檔案總管整合讓 Windows 看起來「本來就有網際網路功能」
- OEM 版本預裝 IE 4.0 讓 Netscape 更難進入新電腦

## 反壟斷的焦點

IE 4.0 的桌面整合直接導致了美國司法部對 Microsoft 的反壟斷訴訟。政府認為 Microsoft 將 IE 與 Windows 非法「捆綁」以維護其桌面作業系統的壟斷地位。1998 年 5 月，司法部正式起訴 Microsoft，IE 的整合策略是訴訟的核心證據之一。
