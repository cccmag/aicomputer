# 雲端運算的普及：從概念到實踐

## 企業採用雲端的轉折點

2010 年被稱為「雲端運算的企業採用元年」。前幾年雲端運算主要是新創公司和開發者社群的玩具——Amazon Web Services 的早期使用者多是 Web 新創公司。但到了 2010 年，財富 500 強企業開始認真考慮將工作負載遷移到雲端。

推動企業採用雲端的因素：

**微軟 Azure 正式商用**：微軟的企業客戶終於有了一個可以信任的雲端平台——由他們長期合作的 Microsoft 提供，支援熟悉的 .NET 和 SQL Server 技術棧。

**AWS 的持續創新**：Amazon 在 2010 年推出了多項新服務——Amazon Elastic MapReduce（基於 Hadoop 的大數據分析）、Amazon Virtual Private Cloud（將雲端資源隔離在虛擬私有網路中）、以及 AWS Identity and Access Management（精細的權限管理）。

**成本效益的證明**：多家企業發布了雲端遷移的成本效益分析——展示了在雲端運行的總擁有成本（TCO）低於自建資料中心。

## 基礎設施即代碼的萌芽

2010 年也是 DevOps 運動和「基礎設施即代碼」概念開始獲得關注的一年。Opscode（後來更名為 Chef）和 Puppet Labs 等設定管理工具公司獲得了大量投資。這些工具讓系統管理員可以用程式碼來管理和配置伺服器叢集——而不是手動 SSH 登入每台機器。

## 雲端原生架構的伏筆

雖然 Docker（2013）、Kubernetes（2014）和 Serverless（2014）等技術還未誕生，但 2010 年的雲端運算生態已經奠定了基礎：虛擬機器、物件儲存、訊息佇列、自動擴展——這些抽象概念讓開發者可以將注意力從基礎設施轉移到應用程式邏輯上。
