# Microsoft Azure 平台即服務

## 開發者優先的雲端平台

2010 年 2 月正式商用的 Windows Azure 不僅是一個基礎設施平台，更是一個完整的開發者生態系統。Microsoft 將 Azure 定位為「在雲端執行 .NET 應用程式的最佳方式」——對於已經投資 Microsoft 技術棧的企業來說，這是一個極具吸引力的價值主張。

Azure 的開發者工具整合：

**Visual Studio 整合**：開發者可以直接從 Visual Studio 建立、除錯和部署 Azure 應用程式。本機模擬器讓開發者可以在自己的電腦上測試雲端應用。

**.NET 服務**：Azure 提供了服務匯流排（Service Bus）、存取控制（Access Control）和工作流程（Workflow）等 .NET 服務的雲端版本。

**PHP、Java 和 Python 支援**：雖然 Azure 最初以 .NET 為中心，但 Microsoft 在 2010 年也提供了對 PHP、Java 和 Python 的基礎支援。

## Azure AppFabric

Azure AppFabric 是 Microsoft 提供的中介服務層——讓企業可以將內部部署的應用程式安全地暴露到雲端。AppFabric 包括了服務匯流排（用於跨越防火牆的通訊）和存取控制（用於聯合身分驗證）。

## 與 AWS 的競爭

2010 年 Azure 與 AWS 的競爭格局：AWS 在功能廣度上領先——提供 S3、EC2、SimpleDB、SQS、CloudFront 等多樣化服務。Azure 則在開發者體驗和企業整合方面具有優勢——特別是對於已經使用 Active Directory、SQL Server 和 Visual Studio 的企業客戶。

兩者在 2010 年的競爭推動了雲端服務價格的持續下降和功能的不斷創新。
