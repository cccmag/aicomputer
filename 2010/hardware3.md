# Microsoft Azure 正式商用

## 從「軟體加服務」到雲端平台

2010 年 2 月 1 日，Microsoft 正式宣布 Windows Azure 平台商用——這是 Microsoft 進入雲端運算市場的戰略核心。Azure 的開發始於 2008 年 PDC 大會上的預覽版，經過一年多的開發和測試後正式上線。

Azure 最初的產品範圍集中在三個核心服務：

**雲端運算（Compute）**：提供 Windows Server 2008 虛擬機器執行個體。開發者可以在雲端執行基於 .NET 的 Web 應用程式和背景服務。

**雲端儲存（Storage）**：提供 Blob（二進位大物件）、Table（結構化資料）和 Queue（非同步訊息）三種儲存模式。與 Amazon S3 和 SimpleDB 形成競爭。

**SQL Azure**：基於 SQL Server 的關係型資料庫雲端服務。

## 平台即服務的策略

與 Amazon Web Services 的基礎設施即服務（IaaS）策略不同，Azure 初期專注於平台即服務（PaaS）。開發者不需要管理虛擬機器、作業系統更新或負載平衡器——只需部署應用程式程式碼，Azure 會自動處理所有基礎設施。

這個策略讓 .NET 開發者可以輕鬆地將現有應用程式遷移到雲端，但同時也限制了執行非 Windows 工作負載的能力。Azure 在 2010 年還不支援 Linux 虛擬機器（Linux 支援在 2012 年才加入）。

## 影響與競爭

Azure 的正式商用意味著雲端運算市場從 Amazon 的獨角戲變成了雙巨頭競爭。Google App Engine（2008 年發布）也在這個市場中佔有一席之地，但無論是功能廣度還是市佔率都無法與 AWS 和 Azure 相比。

Azure 的推出對企業客戶尤其重要——許多大企業與 Microsoft 有長期的軟體授權關係，Azure 提供了一條自然的雲端遷移路徑。
