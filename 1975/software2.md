# CP/M：跨平台作業系統的崛起

## 從 Intellec-8 到商業產品

Gary Kildall 在 1974 年開發的 CP/M（Control Program for Microcomputers）原型，在 1975 年正式以商業產品的形式問世。Kildall 和他的妻子 Dorothy 創立了 Digital Research 公司，開始向電腦製造商和愛好者銷售 CP/M。

CP/M 最初是為 Intel Intellec-8 開發系統設計的，但它的架構——特別是 BIOS 層的設計——讓它能夠輕鬆移植到不同的硬體平台。只要一台機器有 Intel 8080（或相容）處理器和至少 16KB 的 RAM，就可以運行 CP/M。

## 為什麼 CP/M 重要

在 CP/M 出現之前，每一台微電腦都有自己的作業系統（如果有的話），軟體開發者必須為每種機器重新編寫程式。CP/M 改變了這一切——它提供了一個統一的 API 和檔案系統，讓同樣的軟體可以在不同製造商的機器上運作。

CP/M 的檔案系統使用 8.3 檔名格式（主檔名 8 個字元、副檔名 3 個字元），這個格式後來被 MS-DOS 沿用，直到 Windows 95 才逐漸淡出。

## 應用軟體生態系

CP/M 的成功吸引了大量的應用軟體。文字處理軟體（如 WordStar）、試算表軟體（如 SuperCalc）、資料庫系統（如 dBase II）都選擇在 CP/M 上開發。這些軟體反過來推動了 CP/M 的銷售，形成了一個正向循環的軟體生態系。

到 1970 年代末，CP/M 已經成為微電腦作業系統的事實標準，DRI 的年收入超過 500 萬美元。
