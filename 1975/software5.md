# 軟體工程的實務化

## 從手工藝到工程

1975 年，軟體開發正經歷一場深刻的變革。隨著電腦應用從大型機擴展到迷你電腦和微電腦，軟體的規模和複雜度持續增長，傳統的「寫了就跑」方式已經無法應付。

這一年出現了幾個重要的軟體工程發展：

- **結構化程式設計**：經過多年的學術辯論，結構化方法開始被業界廣泛接受。越來越多的組織要求開發者使用結構化控制結構，捨棄 goto 語句。
- **原始碼管理**：隨著程式碼規模增長，版本控制和變更管理的重要性日益凸顯。SCCS（Source Code Control System）由 Bell Labs 的 Marc Rochkind 在 1972 年開發，到 1975 年開始在 Unix 社群中獲得採用。
- **除錯工具**：高階語言編譯器搭配符號除錯器的組合開始普及，取代了傳統的二進位傾印除錯方式。

## 程式設計方法的百花齊放

1975 年，幾種主要的程式設計方法論正在爭奪主流地位：

**結構化設計（Structured Design）**：由 Larry Constantine、Edward Yourdon 和 Glenford Myers 等人發展，強調模組化、高內聚（cohesion）和低耦合（coupling）的原則。

**逐步求精（Stepwise Refinement）**：Niklaus Wirth 的方法，從高層抽象逐步向下分解到具體實作。

**主程式-子程式架構**：這是當時最常見的程式組織方式，由一個主程式呼叫一系列子程式。

這些方法雖然在細節上有所不同，但共同目標都是讓程式更容易理解、更可靠、更容易修改。

## 軟體危機

「軟體危機」（Software Crisis）這個概念仍然主導著 1975 年的討論。大型專案超時、超支、充滿錯誤、難以維護——這些問題在 1968 年的 NATO 軟體工程會議上被首次描述，到 1975 年依然存在。

這場危機推動了軟體工程學科的誕生。大學開始開設軟體工程課程，企業開始建立正式的開發流程和品質管制體系。
