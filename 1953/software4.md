# UNIVAC Short Code — 直譯式程式語言的先驅

一九五三年，UNIVAC I 上的 Short Code 語言仍然是主要的程式設計工具之一。Short Code 由約翰·莫奇利（John Mauchly）於一九四九年開發，是世界上第一個為電子計算機設計的高階語言。雖然它的功能在今日看來極其有限，但 Short Code 在程式語言演化史上的地位不可取代。

Short Code 的設計理念非常明確：讓程式設計師能夠用接近數學表達式的方式來編寫程式。傳統的機器語言要求程式設計師將每個運算分解為最基礎的步驟——載入、加法、儲存。Short Code 允許程式設計師直接寫出「X = A + B」這樣的表達式，由 Short Code 直譯器將其轉換為機器指令。

Short Code 的本質是一個直譯器——一支特殊用途的程式，它讀取用 Short Code 語法寫成的「偽碼」，在執行時將其翻譯為 UNIVAC I 的機器指令。這種直譯式執行模式與今日的 Python、JavaScript 並無本質差異：程式碼被逐行讀取、解析、執行，而不需要事先編譯。

Short Code 支援五十種操作，包括算術運算（加減乘除）、函數運算（平方根、三角函數）、以及流程控制（條件轉移）。程式設計師可以用簡短的符號來表示這些操作，例如「1」代表加法、「2」代表減法、「7」代表平方根。

Short Code 的限制也很明顯。由於當時計算機的記憶體極其有限（UNIVAC I 僅有 1000 個字），Short Code 直譯器本身占據了大量空間，留給使用者程式的空間所剩無幾。此外，直譯式執行帶來了巨大的效能開銷——Short Code 程式的執行速度比等價的機器語言慢五十倍以上。

儘管如此，Short Code 的實驗證明了兩個重要的命題：第一，高階語言確實可行，程式設計師可以用更接近人類思考的方式來編程；第二，高階語言的效能代價是可以承受的（至少在科學計算和商業應用中）。這兩個命題為後來的 FORTRAN、COBOL、ALGOL 等語言提供了重要的經驗證據。

一九五三年，Short Code 的使用者雖然為數不多，但他們已經體驗到了高階語言帶來的生產力提升。這個經驗在兩年後——一九五五年 IBM 推出 FORTRAN 時——得到了更廣泛的印證。從 Short Code 到 FORTRAN，再到 ALGOL、C、Java、Python，程式語言的演化本質上是在不斷提升抽象層次，而這個演化過程的起點之一，就是一九五三年的 UNIVAC Short Code。
