# 貝爾實驗室資料傳輸實驗 — 數位資料在電話線上的傳輸

一九五三年，貝爾實驗室（Bell Labs）正在進行一項當時看來並非當務之急的研究：如何透過電話線路傳輸數位資料。這些實驗雖然未立即產生重大成果，但它們為後來的數據機技術奠定了理論和實踐基礎。

電話網路原本是為類比語音訊號設計的。人聲透過麥克風轉換為連續變化的電壓訊號，經過電話交換機的傳輸，在接收端透過聽筒還原為聲音。但數位資料與語音訊號在本質上截然不同：數位資料是一連串的 0 和 1，是離散的脈衝而非連續的波形。

貝爾實驗室的研究人員探索了將數位脈衝調變為類比訊號的方法。最早的實驗採用了振幅調變（AM）和頻率調變（FM）技術——將數位訊號的 0 和 1 分別對應到不同的載波頻率。這種方法後來被稱為頻移鍵控（Frequency Shift Keying, FSK），是早期數據機的核心技術。

實驗遇到了多重挑戰。電話線路是由語音通訊設計的，其頻率響應不適合高速數位傳輸——高頻訊號會被衰減，低頻訊號可能受到電源雜訊的干擾。此外，長距離電話線路中的中繼放大器會引入失真和延遲，這些在語音通訊中可以容忍，但在數位傳輸中會導致誤碼。

一九五三年，這些實驗還處於非常早期的階段。貝爾實驗室的研究人員只能在實驗室環境中實現每秒幾百位元（bps）的傳輸速率，而且誤碼率相當高。真正的突破要等到一九五八年——貝爾實驗室推出 Bell 101 數據機，實現了每秒一百一十位元的傳輸。

然而，這項研究的重要性在於它提出了「資料通訊需要專門的硬體和協議」這一概念。在過去，資料交換是透過人類搬運打孔卡或磁帶來完成的——這是所謂的「運動鞋網路」（Sneakernet）。貝爾實驗室的實驗預示著：未來的資料交換可以透過電信網路即時進行。

從一九五三年的實驗室實驗到一九六○年代的分時系統、一九七○年代的 ARPANET、一九八○年代的數據機撥接上網、一九九○年代的寬頻網路、以及二○○○年代的行動通訊——這條漫長的技術演化路徑，起點就是貝爾實驗室那些看似不起眼的資料傳輸實驗。

一九五三年的這些研究還有一個更深遠的影響：它使後來的研究人員意識到，數位通訊與類比通訊需要完全不同的理論框架。這個認識直接催生了克勞德·夏農（Claude Shannon）的資訊理論——雖然夏農的論文在一九四八年就已發表，但其在資料通訊中的應用直到一九五○年代才逐漸被理解。
