# Linux 2.2 核心：開放原始碼的里程碑

## 長久等待的結束

1999 年 1 月 25 日，Linus Torvalds 正式釋出了 Linux 核心 2.2.0。這是一次重要的版本躍進，從 2.0 系列到 2.2 系列歷經了三年多的開發。

Linus 在公告中幽默地寫道：「夠了就是夠了，我不能永遠發布 pre-kernel，這只會稀釋了整個概念。」他將最終版本命名為 2.2.0-final，以避免可能出現的難堪拼寫錯誤。

## 重大改進

Linux 2.2 核心引入了多項重要功能：

**IPv6 支援**：首次在 Linux 核心中加入了對下一代網際網路協定 IPv6 的支援，為未來網路的擴充性做準備。

**SPX/IPX 與 DDP 協議**：加入了對 SPX 協定的支援（搭配 IPX），改進了 AppleTalk 網路的 DDP 協定支援。

**改善的 SMP 能力**：對多處理器系統的支援更加成熟，讓 Linux 能夠更好地運用伺服器級硬體的效能。

**AGPgart 核心模組**：加入了對 AGP 顯示介面的支援，改善了圖形效能的底層基礎。

## 產業反應

Linux 2.2 的穩定性和功能豐富度使 Linux 在企業市場中有了更強的立足點。各 Linux 發行版——包括 Red Hat、SuSE、Debian——紛紛在幾個月內推出基於 2.2 核心的新版本。

Corel 也在年底推出了 Corel Linux OS，這是一個基於 Debian 的桌面發行版，顯示了 Linux 從伺服器向桌面市場擴張的野心。
