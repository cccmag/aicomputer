# IBM 1401 程式設計：SPS 與 RPG

1960 年代初期，IBM 1401 的程式設計主要依賴兩套重要的軟體工具：SPS（Symbolic Programming System）組譯器與 RPG（Report Program Generator）報表產生器。這兩套工具分別服務於不同技術背景的使用者——SPS 針對專業程式員，RPG 針對商業分析人員——共同構成了 1401 豐富的軟體生態。

SPS 是 1401 的主要程式開發工具，一套符號組譯器（symbolic assembler）。在 SPS 出現之前，1401 的程式員需要直接用機器碼編寫程式——使用 6 位元的 BCD 指令碼與八進制位址。這是一個極其繁瑣且容易出錯的過程。SPS 引入了符號位址與助憶碼：程式員可以用有意義的標籤來標記位址（如 PAYROLL-START、TAX-CALC），用字母縮寫來取代操作碼（如 B 代替 011010，表示 Branch）。SPS 將這些符號轉換為 1401 可執行的機器碼。

SPS 的翻譯過程分為兩次掃描。第一次掃描（pass 1）讀取原始碼，建立符號表（symbol table），記錄每個標籤對應的位址。第二次掃描（pass 2）使用符號表中的資訊，將每條指令轉換為對應的機器碼，並輸出到卡片或磁帶上。SPS 還提供了條件式組譯（conditional assembly）、巨集指令（macro instruction）與交叉引用列表（cross-reference listing）等進階功能。

RPG 是 IBM 為 1401 開發的一套革命性的報表產生工具。它的目標使用者不是專業的程式員，而是熟悉商業流程但不熟悉程式設計的會計師、稽核人員與業務分析師。RPG 的使用者只需要描述「輸入資料的格式」與「輸出報表的格式」，RPG 編譯器會自動產生完成資料讀取、處理與輸出所需的機器碼。

RPG 程式的結構以規格表（specification forms）為基礎。使用者填寫五種規格表：檔案描述規格（File Description Specifications）、輸入規格（Input Specifications）、計算規格（Calculation Specifications）、輸出規格（Output Specifications）與附加資料規格（Additional Data Specifications）。每個規格表是一個表格，使用者按欄位填入資訊——例如在輸入規格表中指定輸入記錄中每個欄位的名稱、位置、長度與資料類型。

RPG 的計算規格表支援基本的算術運算（ADD、SUB、MULT、DIV）與條件判斷（IF、ELSE、END）。但 RPG 的強項不在於複雜的計算邏輯，而在於其檔案處理能力——它可以同時處理來自卡片、磁帶、磁碟與印表機的資料流，自動進行記錄比對、合併與彙總。一個典型的 RPG 程式——例如產生月度銷售報表——可以在數小時內完成，而用 SPS 或 COBOL 來實現可能需要數天。

IBM 為 1401 提供了豐富的應用程式庫（application library），涵蓋了工資計算、應收帳款、存貨管理、總帳會計等常見商業功能。這些應用程式以 SPS 原始碼或 RPG 規格表的形式提供，客戶可以根據自己的需求進行修改與擴展。IBM 的系統工程師團隊會協助客戶安裝與客製化這些程式。

1401 的軟體生態為 IBM 帶來了巨大的競爭優勢。當競爭對手推出硬體規格相似的產品時，IBM 可以強調其成熟的軟體工具與應用程式庫——這套軟體生態不是競爭對手可以在短期內複製的。這個策略—以軟體差異化來保護硬體市場佔有率—在後來成為 IBM 長期成功的關鍵因素之一。

SPS 與 RPG 這兩套工具代表了 1960 年代初期商業程式設計的兩種路線：SPS 給予專業程式員接近硬體的靈活控制能力，RPG 讓非技術使用者也能參與程式開發。這兩條路線至今仍然存在——前者演化為 C/組合語言，後者演化為 SQL 與第四代語言。
