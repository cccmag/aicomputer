# 歷史穿梭：1967 年的種子

## 物件導向的傳承：從 Simula 到 Java

1967 年 Simula 67 的類別與繼承機制，在接下來的半個世紀中發展為一條清晰的路線：

**Simula 67 (1967)** → **Smalltalk (1972)** → **C++ (1983)** → **Java (1995)** → **C# (2000)** → **Python (物件導向特性)**

物件導向不僅改變了程式語言的設計，也改變了軟體工程的方法論。UML、設計模式（Design Patterns）、重構（Refactoring）等概念都建立在物件導向的基礎之上。

Simula 的發明者 Dahl 和 Nygaard 在 2001 年共同獲得了 IEEE von Neumann 獎章，並在 2002 年獲得了圖靈獎——這是對他們 1967 年工作的遲來但應有的肯定。

## 從兒童程式到運算思維

Seymour Papert 的 LOGO（1967）開創了一條教育科技的新路線：

**LOGO (1967)** → **Smalltalk 的兒童版 (1970s)** → **Scratch (2007, MIT)** → **Blockly (2012, Google)**

Papert 的核心思想——讓孩子成為程式設計師而非被程式設計——在今天比以往任何時候都更加重要。全球的「Hour of Code」活動和運算思維（computational thinking）課程，都可以追溯回 1967 年這隻在 MIT 地板上爬行的機械海龜。

## 微程式設計的統治

Wilkes 的微程式設計（1951 年概念，1967 年圖靈獎肯定）是一條重要的技術傳承：

**EDSAC 2 (1958)** → **IBM System/360 (1964)** → **x86 處理器 (1978 至今)**

微程式設計讓 IBM 可以在不同的硬體實現上運行相同的指令集——這是 System/360 兼容性策略的技術基礎。這種「一次編寫，到處運行」的理念，比 Java 的 JVM 早了三十年。

## 網路的分組交換轉折

1967 年在網路領域發生的三件事——Roberts 的 SOSP 論文、Clark 的 IMP 提議、Davies 的分組交換——分別來自三個不同的方向，卻在歷史的某一點上交會：

**Baran (1960-64) 的軍用分散網路** →  
**Davies (1965-67) 的分組交換** →  
**Clark (1967) 的 IMP 架構** →  
**Roberts (1967) 的 ARPANET 藍圖** →  
**1969 年首次 ARPANET 連線** → **網際網路**

如果缺少任何一個環節，網際網路的歷史都可能完全不同。

## 從浮閘電晶體到固態硬碟

1967 年貝爾實驗室的浮閘電晶體，是儲存技術革命的起點：

**浮閘電晶體 (1967)** → **EPROM (1971, Intel)** → **EEPROM (1978)** → **快閃記憶體 (1984, 東芝)** → **固態硬碟 (1990s)** → **NVMe SSD (2010s)**

這條路線從一個物理學概念，經過幾十年的工程改進，最終徹底改變了電腦的儲存架構——讓行動裝置、輕薄筆電、高速伺服器成為可能。

## 1967 年的時代脈絡

1967 年的科技發展並非發生在真空之中：

- **冷戰高峰**：ARPANET 和網路研究背後的國防經費反映了美蘇對抗的格局
- **反文化運動**：Papert 的教育理念和個人化計算的種子，與當時的個人解放思潮相呼應
- **太空競賽**：IBM 1360 儲存系統和大型主機的發展，與 Apollo 計畫對計算能力的需求直接相關
- **消費社會的崛起**：第一屆 CES 預示了電子產品從軍工到消費市場的重大轉向

1967 年最迷人的地方在於：許多今天我們視為理所當然的概念——物件導向、觸控螢幕、分組交換、軟碟——都是在這一年首次出現的。
