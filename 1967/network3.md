# ASCII 標準化：字符編碼的統一

## 從混亂到秩序

1967 年，ASCII（American Standard Code for Information Interchange）標準正式發布為 ASA X3.4-1963 的修訂版。雖然基礎標準在 1963 年就已制定，但 1967 年的版本將 ASCII 推向了更廣泛的採用。

在 ASCII 之前，電腦製造商各自使用不相容的字符編碼——IBM 的 EBCDIC、BCDIC、以及其他廠商的專用編碼。不同電腦之間交換文字資料是一場噩夢。ASCII 的目標是建立一個通用的字符編碼標準，讓不同系統之間可以互通。

## 7 位元的抉擇

ASCII 使用 7 位元編碼，可以表示 128 個字符，包括：
- 大小寫英文字母（52 個）
- 數字（10 個）
- 標點符號（32 個）
- 控制字符（34 個，如換行、回車、跳格等）

7 位元的選擇在當時是有爭議的：8 位元可以表示 256 個字符，但 7 位元被認為對大多數應用已經足夠，而且可以與當時的電報編碼相容。

## ASCII 的統治時代

ASCII 成為 1960 年代到 2000 年代電腦字符編碼的基礎。它的局限性——無法表示非英文字母——最終導致了 Unicode 的誕生，但 ASCII 的影響至今仍在 Unicode 中（前 128 個碼點完全對應 ASCII）。
