# UNIVAC 組合語言與早期助憶碼系統

一九五六年，UNIVAC 的程式設計環境正經歷從機器語言到符號語言的轉變。雷明頓蘭德公司為 UNIVAC I 與 UNIVAC 1103A 開發了多套早期的助憶碼組合系統，這些系統是組合語言發展史上的重要里程碑。

UNIVAC I 最初使用 C-10 指令集，這是一套設計精巧的六位元指令編碼系統。程式設計師必須將指令以六位元二進位或八進位的形式輸入電腦，使用打孔卡片或紙帶作為輸入媒體。這種程式設計方式極其繁瑣且容易出錯，除錯更是耗時費力的過程。

為了解決這個問題，UNIVAC 開發了早期版本的組合器，允許程式設計師使用簡短的英文縮寫來代表操作碼。例如使用「A」代表加法、「S」代表減法、「M」代表乘法、「T」代表傳送。這些助憶碼雖然簡短，但遠比記憶數位編碼要容易得多。

葛麗絲·霍普（Grace Hopper）在 UNIVAC 的工作對這些發展有著深遠影響。霍普在一九五二年開發了第一部編譯器 A-0，雖然她使用的「編譯器」概念與現代不同——A-0 更像是連結器與載入器的前身——但她的工作證明了程式可以用高於機器碼的層次來撰寫。霍普後來領導了 COBOL 語言的開發。

除了組合器之外，UNIVAC 還發展了 Short Code 直譯器。Short Code 是第一部高階語言的嘗試，允許程式設計師使用類似數學表達式的語法。但 Short Code 是直譯而非編譯的，執行效率遠低於機器碼，因此主要用於教學與原型開發。

這些早期的程式設計工具雖然原始，但建立了解釋器、編譯器與組合器的基本概念框架，為日後的語言處理系統提供了重要的先例。
