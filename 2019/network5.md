# TypeScript 與 JavaScript 生態系的蓬勃發展

## TypeScript 的爆炸性成長

2019 年是 TypeScript 從「選項」轉變為「標準」的一年。根據 npm 的下載統計，TypeScript 的套件下載量在 2019 年增長了超過 60%。多個重量級專案在 2019 年採用了 TypeScript：

- **Vue 3** 完全用 TypeScript 重寫（雖然仍支援 JavaScript API）
- **Angular** 從一開始就建立在 TypeScript 之上
- **Deno** 的核心和標準庫全用 TypeScript 編寫

## 前端的框架戰爭

2019 年的前端框架格局大致穩定，但各自有了重要的發展：

**React 16.8**（2019 年 2 月）引入了 Hooks——這是 React 自 2013 年發布以來最重要的 API 變革。Hooks 讓函式元件可以擁有狀態和副作用，減少了 class 元件的複雜度。

**Vue 3** 在 2019 年處於積極開發階段，下半年進入 RC 階段。其 Composition API 受到了 React Hooks 的啟發，但也保留了 Vue 傳統的 Options API。

**Svelte 3** 在 2019 年發布，採用了全新的編譯器驅動方法——在編譯時將框架代碼移除，產生的原生 JavaScript 體積極小。

## 全端 JavaScript

Node.js 12 於 2019 年 4 月成為 LTS 版本，引入了對 ECMAScript Modules 的實驗性支援（--experimental-modules）。這是向現代 JavaScript 生態系統過渡的重要一步。

Next.js 9 引入了自動靜態優化和動態路由，進一步模糊了靜態網站和動態應用之間的界限。Gatsby 和 Next.js 的競爭推動了靜態網站生成器（SSG）和伺服器端渲染（SSR）技術的普及。
