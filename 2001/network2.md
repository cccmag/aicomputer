# 微軟 Azure 雲端概念的雛形

## 從擴展問題到平台思維

2001 年，微軟的工程師們開始認真思考一個根本性的問題：當企業不再想要自己管理伺服器和基礎設施時，微軟應該如何回應？

這個問題的背後是幾個趨勢的匯聚：

- .com 泡沫破滅後，企業 IT 預算緊縮，租用基礎設施的概念變得有吸引力
- 寬頻網路的普及讓遠端存取服務變得可行
- Google 和 Amazon 已經開始建構大規模的內部雲端基礎設施

雖然微軟在 2001 年還沒有正式啟動 Azure 專案（Project Red Dog 要到 2008 年才開始），但這一年微軟內部開始了一系列關於「網路服務平台」的討論。

## 比爾蓋茲的 .NET 戰略

2001 年，比爾蓋茲提出了微軟的 .NET 戰略——將微軟的產品從用戶端軟體轉向網路服務。.NET 的核心概念包括：

- **XML Web Services**：透過標準化協定（SOAP、XML、HTTP）讓不同系統之間可以互通
- **.NET Framework**：全新的應用程式開發和執行平台
- **Windows Server 系統**：作為網路服務的基礎平台

.NET 戰略雖然不完全等同於後來的 Azure，但它為微軟的「軟體即服務」（SaaS）願景奠定了思想基礎。

## 影響 Azure 設計的關鍵概念

微軟在 2001 年左右形成的幾個概念，後來直接影響了 Azure 的設計：

**自動化資料中心**：大規模伺服器場的管理需要全新的自動化工具

**多租戶架構**：讓多個客戶共享同一基礎設施而不影響安全性和隔離性

**彈性擴展**：根據需求自動調整資源分配

**服務等級協議（SLA）**：為雲端服務提供商業級的可用性保證

## 歷史定位

微軟在 2001 年對雲端的思考還處於非常早期的階段。真正的 Azure 平台要到 2008 年才首次公開，2010 年才商業化。但 2001 年的這些思想種子——加上 2005 年 Ray Ozzie 加入微軟後的推動——為微軟後來在雲端市場的成功奠定了基礎。
