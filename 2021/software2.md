# Rust 進入 Linux 核心：系統語言的典範轉移

## 歷史性的決定

2021 年 4 月，Linux 創始人 Linus Torvalds 宣布，Linux 核心 5.13 將引入對 Rust 語言的實驗性支援。這是在 Linux 核心長達 30 年的歷史中，首次允許使用 C 以外的語言編寫核心驅動程式。

這項工作主要由 Miguel Ojeda（Google 工程師）推動，他在 2020 年的 Linux Plumbers Conference 上首次提出了 Rust for Linux 的提案。Ojeda 和核心團隊開發了 Rust 與 Linux 核心 API 之間的繫結層（bindings），讓 Rust 程式碼可以安全地呼叫核心功能。

## 為什麼是 Rust

Linux 核心的絕大部分是用 C 編寫的——C 賦予了開發者精細的記憶體控制能力，但也是大量安全漏洞的根源。根據 Google 的統計，Android 核心中約三分之二的安全漏洞來自於 C 語言的記憶體安全問題。

Rust 提供了 C 級別的性能，同時透過所有權系統確保記憶體安全：

- **所有權（Ownership）**：編譯器在編譯期確定每個值的生命週期
- **借用（Borrowing）**：防止資料競爭和懸空指標
- **生命週期（Lifetimes）**：確保引用始終有效

Rust 的這些特性可以在編譯期消除整類記憶體錯誤——這是 C 語言無法做到的。

## 影響與展望

Rust 進入核心的意義不僅在於語言本身：

- **安全性提升**：驅動程式——通常是核心中最容易出現漏洞的部分——可以用 Rust 編寫，大幅減少記憶體安全漏洞
- **新開發者**：Rust 的現代工具鏈和較高的安全性吸引了新的開發者參與核心開發
- **生態主流化**：這是 Rust 從系統程式語言「小眾選擇」走向「主流選項」的關鍵一步

## 挑戰

Rust 整合到核心中並非沒有挑戰。主要障礙包括：核心 Rust 繫結層的維護工作、需要支援大量古老的硬體架構、以及核心開發社群的學習曲線。截至 2021 年底，Rust 在核心中的支援仍處於實驗階段，但方向已經明確。
