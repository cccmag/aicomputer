# FLoC 到 Topics API：Google 的廣告追蹤變革

## 第三方 Cookie 的終結

2021 年，Google 持續推進了在 Chrome 中淘汰第三方 Cookie 的計畫——這項被稱為「隱私沙盒」（Privacy Sandbox）的倡議，旨在在用戶隱私和廣告生態系統之間找到平衡。

第三方 Cookie 的淘汰對整個數位廣告產業產生了巨大影響：廣告商、發布商和廣告技術公司長期依賴跨站追蹤來定向廣告和測量轉換。

## FLoC 的發布與爭議

2021 年初，Google 開始測試 FLoC（Federated Learning of Cohorts，聯合學習同群）——一種基於興趣的廣告定向方案。FLoC 的核心思路是：使用者在本地瀏覽器中被分配到一個數千人規模的「同群」（cohort），廣告商根據同群而非個人進行定向。

然而，FLoC 發布後立即遭到了廣泛批評：

- **EFF（電子前哨基金會）**：稱 FLoC 為「一個糟糕的想法」，因為它仍然可以透過瀏覽行為特徵化使用者
- **GitHub、WordPress 等平台**：主動封鎖了 FLoC 追蹤
- **隱私倡導者**指出 FLoC 可能引發新的歧視形式（如基於瀏覽行為的價格歧視）
- **競爭對手**：Edge、Firefox 和 Safari 均表示不支援 FLoC

## Topics API 的轉向

2021 年底，Google 放棄了 FLoC，轉而提出 Topics API——一個完全不同的方案。Topics API 的原則是：瀏覽器根據使用者上週的瀏覽活動推斷出數個「興趣主題」（如「健身」「旅遊」「汽車」），廣告商根據這些主題進行定向。

與 FLoC 的關鍵區別是：
- 主題是人類可讀的（如「寵物用品」而非神秘的 cohort ID）
- 使用者可以看到並刪除特定主題
- 主題列表由 Google 維護，具有明確的邊界和內容審查

## 行業影響

Google 從 FLoC 到 Topics 的轉向反映了隱私保護廣告技術的複雜性——沒有任何技術方案可以在不收集個人資料的情況下完美模擬第三方 Cookie 的廣告效果。數位廣告產業正在告別精細化的個人追蹤時代，轉向更粗粒度但更尊重隱私的方案。
