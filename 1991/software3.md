# Microsoft Visual Basic：視覺化程式設計的誕生

## Ruby 與 BASIC 的結合

1991 年 5 月 20 日，在亞特蘭大舉行的 Windows World '91 展會上，Microsoft 創辦人 Bill Gates 親自演示了 Visual Basic 1.0。這款產品的開發故事始於 Alan Cooper 和他開發的原型工具 Tripod（代號 Ruby），一個可以「繪製」Windows 應用程式介面的視覺化工具。

Microsoft 當時正在為 Windows 3.0 尋找更容易的應用程式開發方式。他們收購了 Tripod 的技術，並將其與為已取消的 Omega 資料庫系統所開發的嵌入式 BASIC 引擎（代號 EB）相結合。結果就是 Visual Basic——第一個將視覺化表單設計器與程式語言整合在一起的商用開發工具。

## 革命性意義

在 Visual Basic 之前，建立 Windows 應用程式需要編寫大量的 C 程式碼，深入了解 Windows 的訊息泵架構，手動處理視窗類別的註冊和訊息分發。Visual Basic 讓開發者可以直接從工具箱中拖曳按鈕、文字框和選單到表單上，然後用 BASIC 語法編寫事件處理程式。

Visual Basic 1.0 包含：
- 視覺化表單設計器（drag-and-drop）
- 屬性視窗（Property Window）
- 程式碼編輯器
- 除錯器
- 可編譯為獨立的 .EXE 檔案

## VBX 生態系統

Visual Basic 最關鍵的創新是 VBX（Visual Basic Extension）介面——允許第三方開發者或開發團隊自行建立自訂控制項，動態連結到 VB 應用程式中。這創造了一個蓬勃發展的第三方控制項市場，從資料庫控制項到圖表控制項應有盡有。

1991 年的 Visual Basic 不僅是一個產品，更是一種示範：它證明了複雜的 Windows 開發可以變得簡單、視覺化和優雅。
