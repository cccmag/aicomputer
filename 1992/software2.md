# 386BSD：第一個自由 Unix 作業系統

## BSD 的解放

1992 年 3 月，William Jolitz 和 Lynne Jolitz 發布了 386BSD 0.1——第一個基於 Intel 80386 架構的完全自由且開源的 Unix 作業系統。386BSD 是從 BSD Net/2 程式碼庫移植到 386 平台的結果。

386BSD 的重要性在於，它讓任何擁有 386 PC 的人都可以執行一套完整的 Unix 系統——而不需要購買昂貴的專屬 Unix 授權（如 SunOS、HP-UX 或 IBM AIX）。

## 與 Linux 的競合關係

386BSD 的發布時間（1992 年 3 月）比 Linux 宣布採用 GPL 授權（1992 年 2 月）稍晚，但兩者幾乎在同一個時間點為 x86 平台帶來了自由的類 Unix 作業系統。

386BSD 和 Linux 的關鍵差異：
- **授權**：386BSD 採用 BSD 授權，允許商業使用而不需公開修改後程式碼；Linux 採用 GPL，要求衍生作品也必須開放
- **開發模式**：386BSD 由 Jolitz 夫婦主導，維護較為鬆散；Linux 由 Torvalds 集中管理，社群協作更有效率
- **核心設計**：386BSD 的程式碼品質較高，但驅動程式支援進展緩慢

## 對後代的影響

386BSD 雖然沒有像 Linux 那樣獲得廣泛採用，但它催生了兩個重要的分支：
- **FreeBSD**（1993 年）：由 386BSD 的用戶和開發者社群分叉出來
- **NetBSD**（1993 年）：專注於可移植性，從 386BSD 分支

FreeBSD 後來成為 Yahoo!、WhatsApp 和 Netflix 等大型服務的底層作業系統。386BSD 的發布證明了微軟作業系統霸權之外還有另一條道路——這條路線至今仍在支撐著網際網路基礎設施。
