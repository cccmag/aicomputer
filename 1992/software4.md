# OS/2 2.0：IBM 的桌面作業系統野心

## 藍色巨人的反擊

1992 年 3 月 31 日，IBM 發布了 OS/2 2.0——一款為 386 以上處理器設計的 32 位元作業系統。OS/2 原本是 IBM 與 Microsoft 的合作專案（1987 年開始的共同開發），但到 1991 年底雙方正式「分手」，IBM 獨自負責 OS/2 的後續開發。

OS/2 2.0 的技術成就令人印象深刻：
- **完全 32 位元**：從核心到應用程式介面都是 32 位元，優於 Windows 3.1 的 16/32 位元混合架構
- **搶先式多工**（Preemptive multitasking）：每個應用程式獨立受 OS 排程，單一程式當機不會拖垮整個系統
- ** Workplace Shell**（WPS）：物件導向的使用者介面，比 Windows 的 Program Manager 更先進
- **MS-DOS 相容性**：可在受保護的 DOS Box 中執行 DOS 應用程式
- **Windows 相容性**：內建 WIN-OS/2 子系統，可直接執行 Windows 3.0 應用程式

## 為什麼 OS/2 輸了

從技術角度來看，OS/2 2.0 在 1992 年發布時比 Windows 3.1 更先進。但它失敗的原因包括：

- **硬體需求高**：需要 386 處理器和 4 MB RAM，而當時多數 PC 仍是 286 或 386SX 搭載 2 MB RAM
- **應用程式太少**：雖然可以執行 Windows 程式，但原生 OS/2 應用程式極少
- **缺乏預裝**：IBM 未能說服 PC 製造商預裝 OS/2
- **市場混亂**：Microsoft 的 Windows 3.1 已經建立了強大的生態系統和開發者社群

OS/2 的技術路線雖然在商業上失敗了，但其設計理念——搶先式多工、物件導向桌面、系統穩定性——後來被 Windows NT 和 Windows 95 所吸收。
