# SAP NetWeaver：企業應用平台

## 企業軟體的整合願景

2002 年，SAP 推出了 SAP NetWeaver——一個全新的企業應用和整合平台。這代表了 SAP 從「提供 ERP 軟體」到「提供企業應用平台」的戰略轉變。

SAP NetWeaver 的核心理念是：企業的 IT 環境通常包含來自不同廠商的異質系統——SAP 自己的應用、非 SAP 的應用、資料庫、前端系統等。NetWeaver 旨在成為連結所有這些元件的「技術骨幹」。

## 平台組成

SAP NetWeaver 2002 年版本包含以下主要元件：

**SAP Web Application Server**：Java 和 ABAP 雙堆疊的應用伺服器，支援 Web 應用程式的開發和部署。

**SAP Exchange Infrastructure (XI)**：企業應用整合（EAI）平台，處理不同系統之間的訊息路由和轉換。

**SAP Business Information Warehouse (BW)**：企業資料倉儲解決方案，支援商業智慧和報告功能。

**SAP Portal**：企業入口網站，提供統一的資訊存取點。

**SAP Mobile Infrastructure**：行動應用的基礎設施——雖然在 2002 年行動應用還不普遍，但 SAP 已經預見了這個趨勢。

## 開放標準

與 SAP 早期的封閉策略不同，NetWeaver 採用了開放的技術標準：

- 支援 Java EE（當時稱為 J2EE）
- 支援 Web Services（SOAP、WSDL、UDDI）
- 支援 XML 和 XSLT
- 支援各種資料庫平台（不僅限於 SAP 自家的 MaxDB）

## 市場定位

NetWeaver 的競爭對手包括 IBM WebSphere、Oracle Fusion Middleware 和 Microsoft .NET。SAP 的優勢在於其龐大的 ERP 客戶基礎——對這些客戶來說，NetWeaver 提供了與既有 SAP 系統最好的整合。

NetWeaver 的推出幫助 SAP 在 2000 年代的企業軟體市場中保持了競爭力，並為後來的 SAP HANA（2011 年）和 SAP Cloud Platform 奠定了基礎。
