# 歷史穿梭：2004 年的種子

## 瀏覽器之戰 2.0

1990 年代第一次瀏覽器之戰以 Netscape 的隕落和 Internet Explorer 的壟斷告終。2004 年 Firefox 1.0 的發布標誌著第二次瀏覽器之戰的開始：

**Firefox (2004)** → **Chrome (2008)** → **Edge (2015)** → **Brave/Arc 等新挑戰者**

Firefox 證明了壟斷是可以被打破的——只要有足夠好的產品和使用者的支持。這場運動推動了 Web 標準的快速演進，最終讓瀏覽器市場從 IE 的「一個版本用五年」進入了 Chrome 的「每六週更新一次」的時代。

## 從 Gmail 到雲端服務

Gmail 的 1GB 儲存在當時看來荒謬地巨大——但這個策略改變了人們對網路服務的期望：

**2004 年以前**：網路郵件是小容量、需刪除、低品質的附屬服務

**2004 年以後**：網路服務應該提供「無限」的資源，使用者不應該為空間而煩惱

這個理念後來延伸為 Google 的「永遠的 Beta」文化和免費服務模式。Gmail 的邀請制也成為了後來的產品發布策略模板——稀缺性創造需求。

## Web 2.0 的遺產

2004 年提出的 Web 2.0 概念預言了接下來十年的網路發展：

**Web 1.0 (1990s)** → **Web 2.0 (2004-2010)** → **行動網路 (2010s)** → **AI 原生 (2020s)**

Web 2.0 的三個核心——用戶生成內容、網路作為平台、集體智慧——成為了 Facebook、YouTube、Wikipedia、Twitter 等平台的共同 DNA。雖然 2004 年的瀏覽器技術有限、寬頻速度緩慢，但 Web 2.0 的藍圖已經完整。

## Podcasting：從 RSS 到串流

Podcasting 的起源說明了兩個重要道理：

1. **技術標準的意外應用**：RSS 的 `<enclosure>` 標籤原本是為部落格附件設計的，但它催生了整個 Podcast 產業
2. **簡單勝於複雜**：iPodder 的第一版只是一個幾十行的 AppleScript，但它解決了一個真實的問題

這條路線從 RSS 音訊到今天的串流媒體巨頭（Spotify、Apple Podcasts）經歷了巨大的變化，但核心模式不變——內容創作者可以直接接觸全球聽眾，無需中介。

## 從 Prescott 的失敗到 Core 的崛起

Intel Prescott 的散熱災難是計算機產業史上最重要的失敗之一：

**Prescott (2004)** → **NetBurst 架構終結** → **Core 微架構 (2006)** → **Core i 系列 (2008)**

Intel 從 Prescott 學到的教訓改變了整個 CPU 產業的方向：
- 時脈速度不是一切（MHz myth 的終結）
- 每瓦效能才是行動時代的關鍵
- 多核心比單核心更快更重要

這個轉向讓 Intel 重新獲得了競爭力——但也給了 AMD Opteron 在伺服器市場的成長機會。2004 年的失敗，讓 2006 年的 Core 2 Duo 成為可能。

## 2004 年的時代脈絡

- **美國總統選舉**：網路成為政治動員和募款的重要工具（Howard Dean 的網路競選）
- **伊拉克戰爭與阿布格萊布監獄照片**：數位影像的影響力——Flickr 等平台讓照片成為全球事件
- **Blog 的黃金時代**：部落格作為「個人媒體」的概念達到高峰，傳統媒體開始衰退
- **亞洲海嘯（2004 年 12 月）**：公民記者和 YouTube 前身的影片分享展現了新聞業的未來
