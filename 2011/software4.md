# GitHub 崛起與社交程式設計

## 程式碼的社交網路

2011 年是 GitHub 從新創公司躍升為開發者基礎設施的關鍵一年。GitHub 在 2008 年由 Tom Preston-Werner、Chris Wanstrath、PJ Hyett 創立，以 Git 版本控制系統為基礎，疊加了一個社交層——開發者可以關注（follow）其他開發者、評論（comment）程式碼、發起請求（pull request）協作。

2011 年，GitHub 的使用者數從年初的約 50 萬成長到年底的超過 100 萬。同年 GitHub 推出了 GitHub Pages、Issue 標籤系統、以及 Organizations 功能，讓團隊協作更加順暢。

## Fork + Pull Request 模型

GitHub 最重要的貢獻是普及了 Fork + Pull Request 的協作模型。傳統的開源貢獻流程繁瑣且不透明，而 GitHub 的 Fork 機制讓任何人都可以複製專案、做出修改，然後只需點擊一個按鈕即可發起 Pull Request。專案維護者可以在 Pull Request 中進行逐行評論和修改建議。

## 開源運動的催化劑

GitHub 極大降低了參與開源專案的門檻。2011 年，越來越多的知名專案遷移到 GitHub，包括 jQuery、Ruby on Rails、Node.js、Linux 核心的鏡像也在此時上線。GitHub 不僅是程式碼代管平台，更成為社交程式設計（Social Coding）運動的核心推動者。
