# C++ 商業化：物件導向的普及

## Stroustrup 的計畫

1985 年 10 月，Bjarne Stroustrup 在 AT&T 貝爾實驗室發表了《The C++ Programming Language》一書；同時，第一個商業化的 C++ 編譯器 Cfront 開始對外銷售。這一年標誌著 C++ 從實驗室研究變成可供商業使用的開發工具。

Stroustrup 的目標是設計一個語言，既能像 C 一樣高效控制硬體，又能像 Simula 67 一樣支援物件導向的抽象。C++ 最初的名稱是「C with Classes」（1983），1985 年正式命名為 C++——其中的 ++ 是 C 語言的自增運算子。

## Cfront 編譯器

Cfront 是一個特殊的編譯器：它將 C++ 程式碼先翻譯成 C 語言，再由 C 編譯器編譯成機器碼。這種方法讓 C++ 可以在任何有 C 編譯器的平台上快速部署。

C++ 的核心物件導向機制包括：類別（class）與繼承（inheritance）、成員函數（member functions）、建構子與解構子（constructors/destructors）、運算子重載（operator overloading）、虛擬函數（virtual functions）。

## 商業化的效應

C++ 的商業化推動了物件導向程式設計的大規模普及。到 1980 年代末期，C++ 已經成為系統程式設計、遊戲開發、金融交易系統等領域的標準語言。Microsoft 在 1990 年代初期將 MFC（Microsoft Foundation Classes）建立在 C++ 之上，進一步推動了 C++ 在 Windows 開發中的主導地位。

C++ 的成功也證明了：物件導向不僅是一個學術理念，更是一個可以解決實際工程問題的實用工具。
