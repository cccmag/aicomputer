# C++ 2.0：物件導向語言的標準化

## 從 C with Classes 到 C++

1989 年，貝爾實驗室的 Bjarne Stroustrup 發布了 C++ 2.0 版本的參考手冊。在此之前，C++ 的發展經歷了以下階段：

- 1979 年：Stroustrup 開始開發「C with Classes」——在 C 語言的基礎上加入 Simula 風格的類別支援
- 1983 年：語言正式命名為 C++（++ 是 C 的遞增運算子）
- 1985 年：C++ 1.0 發布，Stroustrup 出版了《The C++ Programming Language》第一版

C++ 2.0 的發布代表了語言的成熟，它引入了多項重要的新特性。

## 2.0 的新增特性

C++ 2.0 加入了後來成為 C++ 核心特性的元素：

- **多重繼承**：一個類別可以繼承多個父類別（雖然這也帶來了一些設計爭議）
- **抽象類別**：透過純虛擬函數（virtual void f()=0）來定義介面
- **運算子多載**的更靈活控制
- **類型安全的連結**（Type-safe Linkage）

這些特性讓 C++ 成為一種真正適合大規模軟體開發的語言。Stroustrup 的設計哲學是「不強迫程式設計師使用任何特定風格」——C++ 同時支援程序式、物件導向和泛型（透過模板，雖然模板在 2.0 中尚未完全成熟）程式設計風格。

## 產業接受

C++ 2.0 的發布標誌著物件導向程式設計在產業界的正式接受。大型軟體專案——從圖形應用、資料庫系統到遊戲引擎——開始採用 C++ 作為主要開發語言。

C++ 的流行也促使其他語言參考其語法：Java、C# 和 JavaScript 的語法都明顯受到 C++ 的影響。
