# Stack Overflow 的社群知識庫

## 集體智慧的累積

Stack Overflow 的核心理念是建立一個持久、高品質的程式設計知識庫。與傳統論壇不同——同樣的問題被反覆問答、好的答案淹沒在垃圾回覆中——Stack Overflow 的設計讓每個問題都只有一個最佳答案，並透過投票機制讓最佳答案浮到最上方。

## 維基模式的問答

Stack Overflow 最具創新性的設計是讓問題和答案都可以被社群編輯。如果一個答案隨著軟體版本更新而過時了，任何人（只要有足夠的聲譽）都可以更新它。這讓 Stack Overflow 的內容不僅是對話的記錄，更是一個不斷進化的知識庫。

## 標籤與分類

Stack Overflow 的標籤系統讓技術問答可以被精確地分類和搜尋。一個關於「Python 列表推導式」的問題可以被標記為 python、list-comprehension、performance 等標籤。開發者可以關注特定標籤、追蹤特定技術領域的最新問題。

## 對程式設計教育的影響

到 2007 年底，Stack Overflow 雖然剛起步，但它即將改變程式設計師學習和解決問題的方式。傳統的學習路徑——閱讀書籍、查閱文件、參加課程——被「搜尋 Stack Overflow」取代。許多開發者在遇到問題時的第一反應，從翻閱手冊變成了在搜尋引擎中輸入問題並加上「stackoverflow」。
