# GitHub 創立：社交化程式設計平台

## 從 Linus 到 GitHub

2005 年，Linus Torvalds 為了管理 Linux 核心的開發，創建了 Git——一個分散式版本控制系統。Git 解決了 BitKeeper 不再免費提供後的核心開發危機，但它最初的命令列介面對許多開發者來說並不友好。

2007 年，Tom Preston-Werner、Chris Wanstrath 和 PJ Hyett 三人在舊金山創立了 GitHub——一個基於 Git 的程式碼託管平台。GitHub 不僅僅是 Git 儲存庫的雲端託管服務，它在版本控制的基礎上加入了社群網路元素：個人檔案、專案頁面、關注（follow）、星號（star）、分支（fork）和 pull request。

## 社交化程式設計

GitHub 最核心的創新是簡化了 Git 的協作流程。在 GitHub 之前，開源專案的貢獻流程繁瑣且不標準——貢獻者通常需要將修補檔透過郵件寄送給維護者。GitHub 的 pull request 機制讓貢獻者可以提交程式碼修改，維護者可以逐行審閱並討論變更。

Fork（複製）功能讓任何人都可以複製一個專案、進行修改、然後提交 pull request。這大幅降低了參與開源專案的門檻——從「我需要被邀請才能貢獻」轉變為「我可以隨時貢獻，由維護者決定是否採納」。

## 對開源社群的深遠影響

GitHub 在 2007 年的創立不僅改變了程式碼託管的方式，也從根本上改變了開源軟體的社會結構。到 2007 年底，GitHub 雖然還只是數千開發者的小社群，但它的影響力即將指數成長。

GitHub 的成功也得益於 Git 本身的技術優勢——分散式架構讓每個開發者都擁有完整的版本歷史，可以離線工作、本地提交。結合 GitHub 的集中式協作平台，形成了當時最強大的開發工作流程。
