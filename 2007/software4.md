# Hadoop 商業化：大數據的起點（Cloudera）

## 從 Google 論文到 Nutch 到 Hadoop

Hadoop 的故事始於 2003 和 2004 年 Google 發表的兩篇論文：Google File System（GFS）和 MapReduce。Doug Cutting 和 Mike Cafarella 在開發 Nutch 開源搜尋引擎時，意識到他們需要類似 GFS 和 MapReduce 的基礎設施來處理海量網頁資料。

2006 年，Cutting 加入了 Yahoo，並在那裡將 Nutch 中的分散式運算元件獨立出來——這個專案被命名為 Hadoop（以他兒子的填充大象玩具命名）。到了 2007 年，Hadoop 已經在 Yahoo 內部被用於大規模資料處理，運行了超過 1000 個節點的叢集。

## Cloudera 的創立

2007 年，來自 Facebook、Google 和 Yahoo 的幾位工程師——包括 Christophe Bisciglia——意識到 Hadoop 的商業潛力。他們在 2008 年正式創立了 Cloudera，但在 2007 年就已經開始規劃。

Cloudera 的目標是將 Hadoop 從少數網路巨頭的內部工具轉變為可以被一般企業使用的資料平台。他們提供 Hadoop 的商業發行版、技術支援、培訓和諮詢服務——類似於 Red Hat 對 Linux 的商業化模式。

## 大數據時代的序幕

Hadoop 的商業化標誌著「大數據」概念的正式誕生。在此之前，海量資料的儲存與分析是 Google、Yahoo 等網路巨頭的專利。Hadoop 讓一般企業也能夠以低成本建立 PB 等級的資料儲存與分析系統。
