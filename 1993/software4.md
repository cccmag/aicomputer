# Slackware Linux：第一個主流 Linux 發行版

## Linux 的安裝噩夢

1993 年，Linux 核心已經從 0.01 成長到 0.99 版，功能越來越完整。但對於一般使用者來說，安裝 Linux 仍然是一場噩夢：你需要從多個 FTP 伺服器下載核心、系統工具、編譯器和程式庫，然後分別編譯和設定。

## Patrick Volkerding 的解決方案

1993 年 7 月 16 日，Patrick Volkerding 發布了 Slackware Linux 1.0——第一個被廣泛採用的 Linux 發行版。Volkerding 是明尼蘇達州立大學 Moorhead 校區的學生，他的目標是為 Linux 提供一個完整、一致且容易安裝的套件。

Slackware 的概念很簡單：將 Linux 核心、GNU 工具、X Window 系統和各種應用程式打包成一個方便安裝的發行版。Slackware 的安裝過程雖然仍然是命令列方式，但比手動編譯每個套件要簡單得多。

## 設計哲學

Slackware 的設計哲學是「保持簡單」（Keep It Simple）。它不使用複雜的套件管理系統（如 dpkg/rpm），而是使用最直接的 tar+gzip 格式。Slackware 的設定全部透過純文字檔案完成，沒有任何圖形工具。

這種設計讓 Slackware 非常穩定、可靠且「乾淨」——對於想要深入理解 Linux 系統運作原理的使用者來說，Slackware 是最好的學習工具。

## 歷史意義

Slackware 是現存最古老的 Linux 發行版（仍在持續維護中）。1993 年 Slackware 的出現標誌了 Linux 從原始核心向一個真正可用的作業系統的轉變。同年稍晚，Debian（1993 年 9 月）和 Red Hat Linux（1994 年）也開始了它們的歷史。

Slackware 的發布讓更多人嘗試了 Linux，從網路管理員到大學教授，從程式設計師到系統管理員。如果沒有 Slackware 讓 Linux 變得更可接近，開放原始碼運動的發展可能會慢好幾年。
