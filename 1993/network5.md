# HTML 與 IMG 標籤：網頁圖片的開始

## 從純文字到圖文並茂

1993 年，Web 開發者在一個看似簡單但極具爭議的問題上展開了辯論：網頁是否應該內嵌圖片？

當時的 WWW 是純文字的宇宙。在 HTML 的原始設計中，超文字文件應該聚焦於資訊的結構（標題、段落、連結），而圖片則作為外部資源透過連結存取。Berners-Lee 的 NeXT 瀏覽器雖然可以顯示圖片，但將它們放在單獨的視窗中。

## Andreessen 的 IMG 革命

Marc Andreessen 在開發 Mosaic 時，決定做一件違反 HTML 正統的事——在 HTML 中引入 `<IMG>` 標籤，將圖片直接嵌入文字流中。這個決定從未經過正式的標準討論——Andreessen 直接在他的 Mosaic 源碼中實作了它。

`<IMG>` 標籤的最初語法極其簡單：`<IMG SRC="url">`。如果圖片無法載入，瀏覽器就顯示一個破碎的圖示。Andreessen 後來開玩笑說，當時的規範就是他寫的 C 程式碼。

## 標準之爭

`<IMG>` 標籤的引入引發了 W3 社群的激烈討論：

- Tim Berners-Lee 擔心內嵌圖片會破壞 Web 的結構化設計理念
- 有人主張使用專用格式（如 MIME multipart）來打包文字和圖片
- 另一些人想要更通用的 `INCLUDE` 機制

但使用者的選擇壓倒了所有的理論爭論。人們喜歡圖文並茂的網頁。當 1993 年 9 月 Mosaic 的 Windows 和 Mac 版本釋出後，`<IMG>` 標籤迅速成為事實標準。其他瀏覽器（包括後來的 Netscape 和 Internet Explorer）都必須支援它。

## 視覺 Web 的誕生

1993 年的 `<IMG>` 標籤讓 Web 從「科學家分享論文」的工具變成了「任何人都可以建立」的視覺媒體。圖片的引入改變了一切：廣告、新聞、電子商務、社交媒體——這些都需要圖片。

在 1993 年之前線上服務的限制——CompuServe 和 AOL 對圖片的支援極為有限。WWW + `<IMG>` 的組合解放了網路內容的視覺表達能力，使得網頁成為真正的「出版媒體」。
