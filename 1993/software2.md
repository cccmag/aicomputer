# Windows NT 3.1：Microsoft 的企業之作業系統

## 從 OS/2 到 Windows NT

1993 年 7 月 27 日，Microsoft 發布了 Windows NT 3.1——這款產品代表了 Microsoft 在作業系統戰略上的根本轉向。NT 的開發始於 1988 年，當時稱為「OS/2 NT」，由 David Cutler（前 DEC VMS 架構師）領導的團隊負責。

當 Microsoft 與 IBM 的合作關係在 1991 年破裂後，NT 專案被重新定位為 Microsoft 自有先進作業系統的基礎。版本編號 3.1 是為了與 Windows 3.1 保持一致——暗示兩者的使用者介面相似。

## 技術成就

Windows NT 3.1 是從零開始全新打造的 32 位元作業系統：

- **完全 32 位元**：無 DOS 底層、無 16 位元相容包袱
- **搶先式多工**：作業系統控制 CPU 排程，應用程式無法壟斷系統資源
- **可移植性**：支援 Intel x86、MIPS R4000 和 DEC Alpha 三種處理器架構
- **安全認證**：C2 級安全性（美國國防部的安全標準）
- **NTFS 檔案系統**：支援檔案級安全權限、日誌和容錯
- **對稱多處理**（SMP）支援
- **Win32 API**：新的 32 位元應用程式介面

## 硬體需求與市場反應

Windows NT 3.1 對硬體的需求在當時堪稱巨大：需要 12 MB RAM（推薦 16 MB）、至少 75 MB 硬碟空間和 386 以上處理器。當時典型的 PC 只有 4 MB RAM，這限制了 NT 的早期採用。

Windows NT 3.1 並非市場上的巨大成功——1993 年銷售約 30 萬套，遠低於 Windows 3.1。但它建立了 Microsoft 在企業級作業系統市場的信譽，為日後的 Windows 2000 和 Windows XP 鋪平了道路。NT 的技術架構至今仍是 Windows 的基礎——從 Windows 10 到 Windows Server 2025。
