# ASCII 標準：美國資訊交換標準碼

## 混亂的編碼年代

1963 年，美國資訊交換標準碼（ASCII）的第一個正式版本由美國國家標準學會（ASA，即後來的 ANSI）發布。這個 7 位元、128 個字元的編碼系統，從根本上解決了不同電腦系統之間文字資料交換的問題。

在 ASCII 出現之前，電腦製造商各自使用獨特的編碼方案。IBM 的 7000 系列使用 6 位元的 BCDIC 碼（後來演化為 8 位元的 EBCDIC），UNIVAC 使用 Fieldata 碼，電傳打字機使用 5 位元的 Baudot 碼。這些編碼系統之間的不相容性，使得一台電腦的資料幾乎不可能在另一台電腦上直接讀取。資料交換的唯一方式是透過人工重新打孔或列印後重新輸入。

## 標準的設計過程

ASCII 標準的制定過程由 ASA 的 X3.2 委員會負責，參與者包括來自 IBM、AT&T、Teletype Corporation、Control Data 和 Honeywell 等主要電腦和通信公司的代表。標準化過程耗時數年，充滿了不同利益方的折衝和妥協。

7 位元編碼的選擇本身是一個妥協的結果。6 位元編碼（64 個字元）無法容納大小寫英文字母加上數字和標點符號；8 位元編碼（256 個字元）雖然空間充裕，但會增加資料傳輸和儲存的成本。7 位元編碼提供了 128 個碼位，足夠容納 95 個可列印字元（大小寫字母、數字和標點符號）和 33 個控制字元。

## 可列印字元的配置

ASCII 的字元配置反映了打字機鍵盤的布局和電信傳輸的需求。大寫字母 A 到 Z 被分配為連續的碼位（65 到 90），小寫字母 a 到 z 同樣是連續的（97 到 122）。這種設計使得大小寫轉換只需要改變一個位元，非常容易在硬體層級實現。

數字 0 到 9 被分配為碼位 48 到 57。標點符號則根據它們在打字機鍵盤上的位置來安排。可列印字元的順序使得字典排序（lexicographic sorting）可以在硬體層級自然實現——這對資料處理應用來說非常重要。

## 控制字元

ASCII 中的 33 個控制字元專門用於通信控制和設備管理。這些控制字元包括：

NUL（空值，0x00）用於填充空白時間。SOH（標題開始，0x01）和 STX（文字開始，0x02）用於訊息分幀。CR（回車，0x0D）和 LF（換行，0x0A）用於控制印表機和終端機的換行。BEL（響鈴，0x07）用於產生警報音。DEL（刪除，0x7F）用於抹除錯誤的字元。

這些控制字元在 1963 年的設計中主要考慮的是電傳打字機和數據機的通信需求，但它們的影響遠遠超出了這個範圍。現代作業系統中的換行符號（Unix 和 Linux 使用 LF，Windows 使用 CR+LF）直接繼承自 ASCII 的 CR 和 LF 控制字元。

## 歷史意義

ASCII 在 1963 年的標準化是電腦史上最重要的標準化事件之一。它為後續所有形式的資料交換——從磁帶交換到網路通信——提供了共同的基底。

ASCII 對程式設計語言和作業系統的設計也產生了深遠的影響。C 語言的字元處理函數（如 isalpha、isdigit）直接建立在 ASCII 的碼位順序之上。網際網路的電子郵件和網頁設計也都以 ASCII 為基礎——HTML 和 HTTP 協定本身就是 ASCII 編碼的文字。

ASCII 最終演化為 ISO 8859（擴充為 8 位元以容納歐洲語言的額外字元）和 Unicode / UTF-8（將編碼擴展到全球所有書寫系統）。但即使在現代 Unicode 環境中，0 到 127 的基本碼位——即 1963 年定義的 ASCII——在每個 Unicode 編碼方案中都是保持不變的。這使得 ASCII 成為了人類電腦文化中最持久、最廣泛的標準之一。
