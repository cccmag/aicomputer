# CDC 3600：Control Data 的大型主機

## 過渡時期的機種

1963 年，Control Data 公司交付了 CDC 3600 大型主機，這是 Seymour Cray 在設計 CDC 6600 之前的過渡性作品。雖然 CDC 3600 的名氣遠不如後來的 CDC 6600，但它在當時是一台效能卓越的大型主機，在科學計算領域佔有重要地位。

CDC 3600 使用電晶體邏輯電路，而非早期的真空管設計。它的字組長度為 48 位元——比 IBM 7094 的 36 位元更大——這使得它在需要高精度數值運算的科學應用中具有優勢。記憶體使用磁芯技術，最大配置為 262,144 個 48 位元字組（約 1.5 MB）。

## 架構設計

CDC 3600 的中央處理器是一個 48 位元的高速運算單元，配備了多個硬體暫存器和豐富的指令集。它的運算速度約為每秒 200,000 到 300,000 次加法，在科學計算應用中表現出色。

I/O 系統使用稱為「通道」（channel）的專用處理器來管理周邊設備的資料傳輸。通道可以在 CPU 執行運算的同時獨立進行資料讀寫。CDC 3600 的周邊設備包括磁帶機、磁碟機、高速印表機和卡片讀取機。Control Data 為 3600 提供了一系列高效的編譯器——包括 FORTRAN 和 COBOL——使其在商業和科學應用中都有競爭力。

## 與 IBM 的競爭

CDC 3600 的主要競爭目標是 IBM 的 7094。在許多科學計算基準測試中，CDC 3600 在浮點運算和矩陣操作方面的表現優於 7094。這使得 CDC 3600 在政府和大學實驗室中獲得了相當的市場佔有率。

Control Data 的市場策略是專注於 IBM 服務不足的高效能科學計算領域。CDC 3600 正是這個策略的具體實踐——它避開了廣大的商業資料處理市場，而是鎖定需要大量數值計算的研究機構。這個利基市場雖然規模不大，但利潤率較高，且客戶對效能的敏感度遠高於對價格的敏感度。

## 作業系統與軟體

CDC 3600 的主要作業系統是 SCOPE（Supervisory Control Of Program Execution），一個支援多道程式設計的批次處理系統。SCOPE 能夠在記憶體中同時保留多個程式，在一個程式等待 I/O 時自動切換到另一個可執行的程式，提高了 CPU 的利用率。

另一常用的系統是 COS（Chippewa Operating System，以 Cray 的家鄉命名），這是一個更簡潔的作業系統，專為大量批次科學計算任務優化。COS 在 CDC 的大型主機使用者中廣受歡迎。

CDC 3600 也配備了功能強大的 FORTRAN IV 編譯器，其最佳化能力在當時處於領先地位。科學家可以在 CDC 3600 上撰寫 FORTRAN 程式，獲得接近手寫組合語言的執行效能。

## 歷史地位

CDC 3600 在 1963 年提供了當時市場上最高的效能成本比，但它在電腦史上的光芒很快被 1964 年的 CDC 6600 和 IBM System/360 所掩蓋。然而，CDC 3600 在技術上扮演了重要的過渡角色——它讓 Control Data 積累了設計和製造高效能大型主機的經驗，這些經驗直接應用到了 CDC 6600 的開發中。

Seymour Cray 在 CDC 3600 中嘗試的許多設計概念——包括高效能浮點運算單元、獨立的 I/O 通道和最佳化 FORTRAN 編譯器——都在後來的 CDC 6600 中得到了更充分的發展。從這個角度看，CDC 3600 是 Cray 走向超級電腦設計之路上的一塊重要踏腳石。
