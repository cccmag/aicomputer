# AJAX 命名與 Web 開發的新範式

## 一個詞彙的誕生

2005 年 2 月 18 日，適應路徑（Adaptive Path）公司的使用者體驗設計師 Jesse James Garrett 發表了一篇文章「Ajax: A New Approach to Web Applications」。在這篇文章中，Garrett 創造了「Ajax」這個詞——Asynchronous JavaScript + XML 的縮寫。

Garrett 觀察到 Google 的幾個產品（Gmail、Google Maps、Google Suggest）使用了一組共同的技術來創造流暢的使用者體驗。這組技術包括：
- HTML/CSS 用於呈現
- DOM（文件物件模型）用於動態顯示和互動
- JavaScript 用於客戶端邏輯
- XMLHttpRequest 用於非同步伺服器通訊
- XML 或 JSON 用於資料交換

這些技術單獨看來都不新——XMLHttpRequest 早在 1999 年 Internet Explorer 5 中就已存在。但它們的組合使用方式才是真正的創新。

## 概念的力量

AJAX 這個詞之所以重要，不在於它引入了新技術，而在於它為一組既有的技術找到了一個容易被記住、容易傳播的名稱。在 AJAX 命名之前，開發者已經在使用這些技術，但沒有一個統一的名詞來描述這個開發模式。

命名讓模式變得可見：一旦 AJAX 被命名，開發者就意識到這是一種新的 Web 開發範式，而非零散的技術拼湊。AJAX 成為 2005-2006 年最熱門的 Web 開發詞彙，催生了大量的框架和工具庫（Prototype、jQuery、Dojo、MooTools 等）。

## AJAX 之外的影響

AJAX 對 Web 開發的深層影響在於：它打破了 Web 應用與桌面應用之間的界線。使用者開始期望 Web 應用具有與桌面軟體相當的互動性和回應速度。這個期望推動了瀏覽器技術的快速演進——JavaScript 引擎的加速（V8、SpiderMonkey）、CSS 的增強、HTML5 的到來，都可以追溯回 2005 年 AJAX 所設定的使用者體驗標準。
