# Web 標準運動：從 IE-only 到跨瀏覽器

## 標準運動的背景

2005 年，Web 開發者仍然深受 Internet Explorer 6（2001 年發布）的困擾。IE6 支援的 CSS 和 HTML 標準遠遠落後於 Firefox，且 Microsoft 將 IE 視為 Windows 平台的一部分而非獨立產品——IE 的更新綁定於 Windows 的版本更新。

Web 標準運動的核心訴求是：瀏覽器應該正確地遵循 W3C 制定的網頁標準，而不是自行其是。開發者希望可以寫一次程式碼，在任何瀏覽器上都能正確呈現。但在 IE6 獨大的時代，這只是一個夢想。

## WaSP 與標準推廣

Web Standards Project（WaSP）成立於 1998 年，由 Jeffrey Zeldman、Glenn Davis 等著名 Web 設計師發起。2005 年是 WaSP 影響力的高峰——他們成功遊說了多家瀏覽器廠商改善標準支援，並教育了 Web 開發社群關於標準的重要性。

2005 年，Firefox 的崛起提供了市場力量來推動標準。開發者開始要求網站支援 Firefox——「在 Firefox 中測試」成為常見的開發流程。這對 Microsoft 形成了壓力，最終促成了 IE7 的開發（2006 年發布，IE6 發布後五年才有的重大更新）。

## 標準的演化

2005 年，幾個重要的 Web 標準正在演化中：
- **CSS 2.1**：在 CSS 2 的基礎上進行了澄清和修正
- **ECMAScript 4**：JavaScript 的下一代標準，雖然最終未能通過，但其概念影響了後來的 ES6
- **WHATWG 成立**：2004 年，Apple、Mozilla、Opera 脫離 W3C 成立了 WHATWG（Web Hypertext Application Technology Working Group），主導 HTML5 的開發——因為他們認為 W3C 偏離了 Web 的實際發展方向

## 深遠影響

Web 標準運動改變了 Web 開發的本質。從 2005 年開始，「跨瀏覽器相容性」從一個專業技能變成了每個 Web 開發者的基本要求。標準化的 CSS 和 JavaScript 減少了開發成本，提高了 Web 應用程式的可靠性，為後來豐富的 Web 應用生態系統奠定了基礎。
