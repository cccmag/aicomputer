# ALGOL 58 發表：國際合作的程式語言

1958 年在瑞士蘇黎世召開的一次會議中，來自美國和歐洲的頂尖電腦科學家共同提出了一種新的程式語言設計方案，這份方案在 1959 年以「國際代數語言」（International Algebraic Language）的名義正式發表，後來被稱為 ALGOL 58。這是最早的國際合作程式語言設計項目之一。

參與 ALGOL 58 設計的科學家包括 John Backus（IBM，FORTRAN 之父）、Peter Naur（丹麥）、Friedrich Bauer（德國）、Alan Perlis（卡內基理工學院）和 Heinz Rutishauser（瑞士）等。他們的目標是創建一種既精確又通用的代數語言，能夠跨越不同的電腦平台進行科學計算。

ALGOL 58 在語言設計上引入了多項創新概念：形式化的語法描述方式（後來發展為 BNF 範式）、區塊結構（block structure）、條件語句的 if-then-else 結構、陣列宣告和巢狀迴圈等。這些概念中的許多後來成為現代程式語言的基本特徵。ALGOL 58 還對變數的型別和作用域進行了嚴格的定義，提高了程式的結構化和可讀性。

ALGOL 58 儘管被稱為「代數語言」，但它的設計並不限於純粹的數值計算。語言規範包含了輸入輸出操作的標準化定義，雖然這部分在後來的 ALGOL 60 中被簡化掉了。ALGOL 58 的報告中還包括了對遞迴（recursion）的討論，雖然當時的實作尚未完全支援。

ALGOL 58 的歷史意義不僅在於語言本身，更在於它所開創的國際合作模式。這種由各國科學家共同參與的標準化過程，為日後程式語言的標準化工作（如 Pascal、C、Java 等）樹立了典範。ALGOL 58 的直接後繼者 ALGOL 60 在 1960 年正式發布，對整個計算機科學領域產生了深遠的影響。事實上，ALGOL 系列語言的語法設計和結構化概念透過 Pascal、C 和 Java 等語言一直影響到今日的軟體開發實踐。
