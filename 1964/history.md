# 歷史穿梭：1964 年的種子與未來的大樹

## System/360 相容概念：從大型主機到 x86

IBM System/360 在 1964 年確立的「同一架構、多種型號、向上相容」概念，成為電腦產業最具影響力的商業模式：

- **1964**：System/360（五個型號、50 倍效能範圍）
- **1970**：System/370（新一代但保持架構相容）
- **1978**：Intel 8086（x86 架構的起點）
- **1981**：IBM PC（採用 x86 架構）
- **1985-2020s**：x86 從 16 位元演化到 64 位元，保持相容

Intel 的 x86 架構是 System/360 相容概念最成功的繼承者。80486 和 Pentium 處理器可以執行 8086 的程式碼，正如 System/360 Model 75 可以執行 Model 30 的程式碼。當今天的 Intel Core i9 處理器啟動時，它仍然以 16 位元的 8086 模式開始執行——這是 1964 年相容概念的延續。

## 從 BASIC 到全民程式設計

1964 年 BASIC 在達特茅斯的誕生，開創了「讓每個人都能寫程式」的傳統：

- **1964**：BASIC 誕生於達特茅斯
- **1975**：Altair BASIC（微軟的第一個產品）
- **1977**：Apple II 內建 AppleSoft BASIC
- **1981**：IBM PC 內建 BASICA（由微軟提供）
- **1991**：Python 誕生（強調可讀性與易學性）
- **2007**：Scratch（MIT 的圖形化程式語言）
- **2013**：Code.org 發起「Hour of Code」

BASIC 的「低學習門檻 + 即時反饋」設計哲學，被後來每一個成功的教育用程式語言所繼承。當 2025 年的一位中學生用 Python 寫出第一行 `print("Hello, world!")` 時，他正在重複 1964 年達特茅斯學生的體驗——只是工具從 Teletype 終端機換成了現代 IDE。

## OS/360 與軟體工程的覺醒

OS/360 的開發經驗——100 萬行程式碼、2 年延遲、數億美元的超支——催生了軟體工程這個學科：

- **1964-68**：OS/360 開發（Brooks 的磨難）
- **1975**：《人月神話》出版
- **1980s**：軟體工程成為大學科系
- **1990s**：敏捷方法開始醞釀
- **2001**：敏捷軟體開發宣言

Brooks 的「沒有銀彈」（No Silver Bullet）論斷——認為軟體開發的本質複雜性無法被任何技術突破所消除——在隨後的數十年中不斷被驗證。人工智慧輔助程式設計（如 GitHub Copilot）可能最終挑戰這個觀點，但 OS/360 留下的教訓仍然適用：大規模軟體系統的開發不僅是技術問題，更是管理問題。

## SABRE 與即時交易系統

SABRE 在 1964 年展示的即時交易處理能力，奠定了現代電子商務的技術基礎：

- **1964**：SABRE 上線（11,000 終端機）
- **1969**：ATM 自動櫃員機開始部署
- **1970s**：銀行業 SWIFT 網路
- **1980s**：電子資料交換（EDI）
- **1994**：Netscape 推出 SSL 安全協定
- **1995**：Amazon 和 eBay 創立
- **2007**：智慧型手機行動支付

SABRE 證明了三件事：電腦網路可以處理即時交易、系統可以全年無休運作、以及自動化可以帶來顯著的商業效益。這三個教訓直接指導了後續所有線上交易系統的設計。

## CDC 6600 到超級電腦與 GPU

Seymour Cray 在 CDC 6600 中展現的設計哲學——以極致工程追求極致效能——成為超級電腦的 DNA：

- **1964-65**：CDC 6600（3 MFLOPs，10 PPU）
- **1969**：CDC 7600（10 MFLOPs）
- **1976**：Cray-1（160 MFLOPs，向量處理）
- **1985**：Cray-2（1.9 GFLOPs）
- **1993**：Intel Pentium（桌上型電腦突破 100 MFLOPs）
- **2007**：NVIDIA CUDA（GPU 通用運算）

Cray 的「多處理器協同」概念在 50 年後以 CUDA 的形式重新出現——數千個 GPU 核心同時運算，正如 CDC 6600 的 10 個 PPU 各司其職。從 CDC 6600 到 TOP500 排行榜上的 Exascale 超級電腦，這條追求「更快」的路線從未間斷。

## ASCII 與資料互通

1964 年 ASCII 標準的推動，為不同電腦系統之間的資料交換提供了共同的語言：

- **1963**：ASCII 草案
- **1964-67**：ASCII 逐步採用
- **1969**：ARPANET 使用 ASCII 作為網路協定的字元編碼
- **1980s**：IBM PC 採用 ASCII（而非 EBCDIC）
- **1993**：UTF-8 誕生（ASCII 的超集）
- **2000s**：全球資訊網全面採用 UTF-8

ASCII —> ISO 8859 —> Unicode / UTF-8 的演化路徑，使得全球的電腦系統最終能夠使用統一的字元編碼。這個標準化過程始於 1964 年，而如果沒有這個起點，全球化的網際網路——一個需要跨國、跨語言、跨系統交換資料的網路——將難以實現。

## 1964 年的時代背景

1964 年是一個政治與文化激烈變動的年代。美國民權法案在該年通過，廢除了法定的種族隔離制度。東京舉辦了夏季奧運會，日本展示了戰後重建的成果。Beatles 席捲美國，引發了「英倫入侵」的文化風潮。

資訊科技的進展嵌入在這些更大的歷史敘事中。System/360 代表了大型企業對效率的追求，BASIC 反映了教育民主化的訴求，SABRE 展現了消費資本主義自動化的趨勢。1964 年的每一項技術突破，都不只是工程師的成就，也是整個社會變遷的縮影。
