# BASIC 語言的誕生：為初學者設計的程式語言

## 數學家對程式設計的反思

1964 年 5 月 1 日清晨，達特茅斯學院的數學系主任約翰·凱梅尼（John Kemeny）和教授湯瑪斯·庫爾茨（Thomas Kurtz）啟動了他們共同設計的程式語言 BASIC 的第一個實作。這個凌晨時分在奇異公司 GE-225 電腦上成功編譯的第一行程式碼，開啟了電腦教育的新時代。

凱梅尼和庫爾茨的動機來自一個簡單的觀察：他們發現達特茅斯學院的學生——特別是那些非科學專業的學生——對學習電腦程式設計有著濃厚的興趣，但他們被當時的程式語言擋在門外。FORTRAN 的語法對初學者來說過於複雜，COBOL 則過於冗長，而 ALGOL 需要理解抽象的形式語法。

凱梅尼曾說：「電腦不應該只是數學家和工程師的專利。如果我們相信電腦將會成為人類社會不可或缺的工具，那麼每個人都應該能夠學習使用電腦——而不必先成為數學家。」

## 語言設計理念

BASIC 的設計圍繞著三個核心原則。第一是簡單性：語法規則必須最小化，讓初學者可以在幾分鐘內開始寫程式。第二是互動性：使用者應該能夠立即看到程式的執行結果。第三是友善性：錯誤訊息必須以易於理解的方式呈現。

這些原則體現在語言的每個細節中。BASIC 的關鍵字都是完整的英文單詞——LET、PRINT、INPUT、IF、THEN、FOR、NEXT、GOTO——而非 FORTRAN 的縮寫和特殊符號。變數不需要事先宣告型態——變數名稱結尾的 $ 號標示字串變數，其他則為數值變數。程式行號的設計讓除錯和修改變得直觀——在兩行之間插入一個新行號即可。

與 FORTRAN 的「先撰寫完整程式、提交執行、等待結果」的模式不同，BASIC 被設計為與 DTSS 分時系統緊密配合。使用者在電傳打字機上輸入一行程式碼，按下 Enter，系統立即檢查語法並回饋結果。這種「即時反饋」的循環是 BASIC 成功的關鍵。

## 第一個程式

達特茅斯學院的 BASIC 使用者手冊中的第一個範例程式，用四行程式碼完全展現了語言的哲學：

```
10 INPUT A, B
20 LET C = A + B
30 PRINT C
40 END
```

這段程式碼沒有任何多餘的語法元素。`INPUT` 從鍵盤讀取數字，`LET` 執行加法，`PRINT` 顯示結果，`END` 標示程式結束。不需要 FORTRAN 的 DIMENSION 宣告，不需要 COBOL 的 DATA DIVISION 結構，不需要 ALGOL 的 BEGIN-END 區塊。

## 從達特茅斯走向世界

BASIC 在 1964 年上線後立即在達特茅斯校園內流行起來。到 1964 年底，達特茅斯的學生已經寫了數萬個 BASIC 程式。凱梅尼和庫爾茨決定將 BASIC 的實作細節公開發表，並積極協助其他大學安裝 BASIC 系統。

這個開放的決策對 BASIC 的傳播至關重要。到了 1965 年，美國東北部的十幾所大學已經安裝了 BASIC 系統。BASIC 的影響力在 1970 年代達到高峰——比爾·蓋茲和保羅·艾倫在 1975 年為 Altair 8800 開發的微電腦 BASIC，催生了微軟公司，也讓 BASIC 成為個人電腦時代最重要的程式語言之一。

BASIC 的遺產不僅僅是它教會了數百萬人如何寫程式。它的設計哲學——讓程式語言適應人類而非讓人類適應語言——影響了後來的 Python、Scratch 以及各種教育用程式語言和工具。
