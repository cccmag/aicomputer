# CDC 6600：Seymour Cray 的超級電腦

## 一個天才的構想

1964 年，當 IBM 忙著推出 System/360 時，位於明尼蘇達州小城市 Chippewa Falls 的一間實驗室裡，Seymour Cray 正在完成他職業生涯中最重要的一項設計——CDC 6600。這台電腦雖然要到 1965 年才正式交貨，但其架構設計在 1964 年就已經定型，並在《Datamation》雜誌等媒體上引起廣泛討論。

Cray 在設計 CDC 6600 時面臨了一個基本的問題：如何讓電腦的運算速度大幅超越所有現有的機器？他的答案是根本性地重新思考電腦的組織方式。

## 十個周邊處理器的創新

CDC 6600 最具革命性的設計特點是它使用了 10 個獨立的周邊處理器（Peripheral Processor Unit，PPU）。這些 PPU 各自擁有自己的 4,096 個 12 位元字的記憶體，負責處理所有 I/O 和中斷服務。中央處理器（CPU）不再需要花時間等待磁帶倒帶或列印完成——這些工作全部由 PPU 包辦。

在 CDC 6600 之前的電腦中，中央處理器必須直接處理 I/O 和中斷，這浪費了大量寶貴的運算時間。Cray 的解決方案相當於在 CPU 周圍部署了一組專職的助理，讓 CPU 可以專注於它最擅長的工作——高速運算。

這個架構與 IBM System/360 的通道（channel）概念有相似之處，但 Cray 的設計更為激進：PPU 是通用的可程式處理器，而非固定功能的通道控制器。這意味著 PPU 可以動態調整它們的工作內容，適應不同的 I/O 需求。

## 機櫃與散熱

CDC 6600 的機櫃設計在 1964 年就展現了 Cray 對機械和美學的極致追求。機櫃採用鋁合金框架，側面板安裝了大型的 Freon 壓縮機冷卻系統。Cray 本人親自設計了機櫃的結構，並堅持使用了他偏好的米色色調。

為了最小化信號傳輸延遲，Cray 將電路板密集地排列在機櫃中，並採用了「三維架構」：中央處理器的電路板安裝在一個十字形的框架上，四周是 PPU 和記憶體模組。這種佈局使得所有關鍵信號路徑都保持在最短的距離。

## 與 IBM 的緊張關係

CDC 6600 的出現引發了 IBM 的強烈反應。當 Thomas Watson Jr. 得知 CDC 6600 的效能數據時——3 MFLOPs，是當時 IBM 最快電腦的數倍——他立即下令 IBM 實驗室必須開發出回應產品。這個命令促成了 IBM 的 ACS（Advanced Computing Systems）部門的成立。

IBM 與 Control Data 之間的競爭在 1960 年代後期更加激烈。1968 年，CDC 對 IBM 提起了反壟斷訴訟，指控 IBM 在電腦市場的壟斷行為。這場訴訟在 1973 年以和解告終，IBM 向 CDC 支付了數億美元，並出售了一些資產。

## 對後世的影響

CDC 6600 所奠定的「主處理器 + 專用輔助處理器」架構，在後續的電腦設計中反覆出現。Cray 後來設計的 Cray-1（1976）雖然使用了向量處理技術，但其多處理器架構的理念部分源自 CDC 6600。現代 GPU 運算中的主機 CPU + GPU 架構、以及現代伺服器中的智慧型 I/O 控制器，都可以追溯到 CDC 6600 的 PPU 設計。

Seymour Cray 在 CDC 6600 中展現的設計哲學——不受既有思維束縛、以極致效能為唯一目標——成為超級電腦設計的不朽精神。
