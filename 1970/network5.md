# RFC 機制的確立：網路標準的民主化

## 一個簡單的想法

1969 年 4 月 7 日，UCLA 的研究生 Steve Crocker 發出了第一份 RFC（Request for Comments）文件——RFC 1：「Host Software」。當時 Crocker 正在協調 ARPANET 不同站點間的軟體開發，需要一個簡單的文件交換機制來記錄技術決策。

Crocker 後來說：「我們這群研究生不想假裝自己是一個正式的標準組織，所以我們把文件稱為 Request for Comments——邀請大家來提意見。」

## RFC 的運作方式

RFC 機制有幾個簡單但強大的原則：
- **開放性**：任何人都可以撰寫和提交 RFC
- **共識**：標準透過討論和共識形成，而非由上而下的命令
- **可演化**：舊的 RFC 可以被新的 RFC 取代
- **編號系統**：每份 RFC 獲得一個永久編號

到 1970 年，RFC 機制已經確立為 ARPANET 開發團隊的主要溝通方式。關鍵的 RFC 包括：
- RFC 1（Host Software）
- RFC 33（New HOST-HOST Protocol）
- RFC 36（Protocol Notes）

## 對網路發展的關鍵作用

RFC 機制造就了網際網路最獨特的文化：一個由技術社群而非中央機構驅動的標準化過程。這種「粗略共識和可執行程式碼」（rough consensus and running code）的哲學，使網際網路能夠快速演化並適應新的需求。

Jon Postel 在 1970 年代後期成為 RFC 編輯，他持續擔任這個角色直到 1998 年去世。在他的管理下，RFC 系列從幾十份文件成長為數千份的網際網路標準庫。
