# 歷史穿梭：1978 年——應用軟體的元年

## 殺手級應用的誕生

1978 年是「殺手級應用」（killer application）這個概念開始具體化的一年：

**VisiCalc (1978 構想, 1979 出貨)** → **Lotus 1-2-3 (1983)** → **Excel (1985)** → **Google Sheets (2006)**

VisiCalc 讓商務人士願意為一台個人電腦花費數千美元——僅為了執行一個程式。它證明了硬體是被軟體推動的。沒有 VisiCalc，Apple II 的普及速度可能會慢得多；沒有 Apple II 的成功，IBM 可能不會在 1981 年推出 IBM PC。

## 從 XMODEM 到現代的檔案傳輸

Ward Christensen 在 1978 年發明的 XMODEM 是一條清晰的傳承路線：

**XMODEM (1978)** → **YMODEM (1980)** → **ZMODEM (1985)** → **FTP (1971 之前)** → **HTTP (1991)** → **BitTorrent (2001)**

XMODEM 引入的「區塊 + 檢錯 + 重送」模式幾乎是所有後續檔案傳輸協定的基礎。

## BBS 的平民連結

CBBS 開啟的平民網路革命：

**CBBS (1978)** → **FidoNet (1984)** → **AOL (1985)** → **WWW (1991)** → **社群媒體 (2000s)**

有趣的是，CBBS 的誕生方式——兩個人在大風雪中決定「自己來做」——正是網際網路未來發展的寫照：去中心化、自發性、由下而上。

## 程式驗證的傳承

Robert Floyd 的歸納斷言方法開創了一條重要的理論路線：

**Floyd 歸納斷言 (1967)** → **Hoare 邏輯 (1969)** → **Dijkstra 的 weakest precondition (1975)** → **模型檢查 (1980s)** → **SMT solver (2000s)** → **形式驗證在工業中的應用 (2010s)**

今天 Intel 的處理器設計驗證、微軟的 Windows 驅動程式驗證、亞馬遜 AWS 的基礎設施驗證，都使用了 Floyd 奠基的形式化方法。

## 8086 的六十年

Intel 8086 是電腦史上最成功的「臨時替代品」：

**8086 (1978)** → **8088 (1979)** → **80286 (1982)** → **80386 (1985)** → **80486 (1989)** → **Pentium (1993)** → **Core 架構 (2006)** → **現代 x86 (2020s)**

x86 架構的壽命遠遠超過了任何人的預期。iAPX 432 早已被遺忘，但 8086 的血脈仍然在每一台 PC、伺服器甚至超級電腦中跳動。

## VT100 的終端機遺產

VT100 定義的 ANSI 逸出序列已成為 Unix/Linux 終端機的事實標準：

**VT100 (1978)** → **xterm (1984)** → **GNOME Terminal (2000s)** → **iTerm2 (Mac, 2000s)**

即使到了 2020 年代，你仍然可以在 macOS 的終端機中輸入 `echo -e "\e[31mHello\e[0m"` 來看到紅色的文字——這是一個 1978 年 VT100 的標準序列。

## 1978 年的時代脈絡

- **經濟壓力推動自動化**：1970 年代末的經濟停滯迫使企業尋找提高生產力的方法，電腦成為解決方案
- **晶片價格持續下降**：Intel 8086 賣 86.65 美元，處理器價格已經低到可以放進各種設備
- **電話網路的開放**：Carterfone 決策（1968）讓非 AT&T 設備可以合法連接電話網路，為 BBS 和數據機市場鋪路
- **軟體商業化**：MicroPro、Personal Software 等純軟體公司的出現，預示了軟體產業的獨立

1978 年最值得記住的是：這一年人們不再問「電腦能做什麼？」，而是開始問「我想用電腦做什麼？——然後把它寫出來」。
