# Subversion 與 Git：版本控制的競爭

## Subversion 的主流地位

2008 年，Subversion（SVN）仍然是企業和開源專案中最廣泛使用的版本控制系統。SVN 採用中央集中式架構——所有程式碼儲存在一台中央伺服器上，開發者從伺服器取出（checkout）最新版本，修改後提交（commit）回去。

SVN 的優勢在於簡單：學習曲線平緩、概念直觀（就是一個檔案伺服器）、權限管理完善。但它的核心限制也逐漸顯現：無法離線提交、分支合併操作複雜且效能差、高度依賴中央伺服器的可用性。

## Git 的崛起

Git 在 2005 年由 Linus Torvalds 創建，但直到 2007 年 GitHub 創立後才開始獲得廣泛關注。2008 年，越來越多的開源專案開始從 SVN 遷移到 Git。

Git 的核心優勢來自分散式架構：

**每個開發者擁有完整歷史**：git clone 不僅僅是取出最新版本，而是複製整個儲存庫的完整歷史。開發者可以在離線時提交、建立分支、檢視歷史——所有操作都不需要網路。

**輕量級分支**：在 Git 中建立分支幾乎是即時的。分支只是一個指向特定提交的指標。這讓「建立分支進行實驗」變成了一種日常操作，而非重大決策。

**合併追蹤**：Git 內建了強大的合併機制，自動追蹤分支之間的關係。多次合併不會產生衝突——這在 SVN 中是一個常見的痛點。

## 競爭的結果

2008 年是 Git 從 RPG 玩家社群的工具轉變為主流程式設計工具的一年。GitHub 的社交功能——fork、pull request、關注——讓 Git 的學習曲線變得值得。到 2008 年底，越來越多的知名專案（如 Ruby on Rails、Linux 核心、Android）都在 Git 上開發。
