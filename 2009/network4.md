# 大數據概念的崛起

## 從資料庫到資料湖

2009 年，「大數據」（Big Data）這個詞開始從技術小圈子進入主流商業用語。這個概念的興起得益於幾個技術趨勢的匯合：數位化資料的爆炸性增長、Hadoop 生態系統的成熟、以及雲端運算提供的低成本儲存與運算能力。

## Hadoop 生態系統的成熟

2009 年 Hadoop 相關專案快速增加：

**HBase**：在 Hadoop 之上執行的分散式資料庫，類似於 Google 的 Bigtable。提供對大規模結構化資料的隨機即時讀寫。

**Hive**：將 SQL 查詢轉換為 MapReduce 任務的資料倉儲工具——讓熟悉 SQL 的分析師可以直接查詢 Hadoop 叢集中的資料。

**Pig**：一種高階的資料流腳本語言，用於簡化 MapReduce 的開發。

**ZooKeeper**：分散式協調服務，管理叢集中的設定同步和服務發現。

## 大數據的產業化

2009 年，Cloudera 開始提供 Hadoop 的商業發行版和企業支援。除了 Cloudera 之外，新的 Hadoop 相關新創公司紛紛成立——MapR（2009）、Hortonworks（2011）——形成了 Hadoop 生態系統的商業版圖。

大數據的概念也超越了 Hadoop 本身。傳統資料庫廠商（Oracle、IBM、Microsoft）開始推出大數據解決方案。資料科學家的角色開始出現——結合統計學、機器學習和程式設計技能來從數據中提取商業洞察。
