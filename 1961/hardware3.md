# IBM 1401 普及：會計機器的電子化替換

1961 年，IBM 1401 開始大量出貨給全球各地的企業與政府機構，展開了大規模的市場滲透。這款在 1960 年發表的電晶體化商用電腦，在 1961 年證明了傳統的卡片穿孔會計機（EAM）大量被電子電腦取代的時代已經來臨。

IBM 1401 的設計哲學可以追溯到 IBM 407 等機電式會計機。IBM 407 是一台透過插線板程式化的機器——操作員需要物理地將電線插入插線板的不同孔位來定義資料流與計算邏輯，修改程式意味著更換整塊插線板。1401 則將程式儲存在記憶體中，可以透過讀入卡片或磁帶上的新程式來切換功能，不僅靈活度大幅提高，而且運算速度也快了數十倍。

1401 的銷售成功有多方面的因素。首先是價格：IBM 將 1401 的月租金定在 2,500 至 5,000 美元之間（依配置而異），低於大型主機的租金，正好落在中型企業的 IT 預算範圍內。其次是周邊系統的完整性：IBM 為 1401 提供了完整的周邊設備組合，包括卡片讀卡機/穿孔機（1402）、高速印表機（1403，每分鐘 600 行）、磁帶機（7330/729）、以及磁碟機（1311，1962 年推出）。

最重要的是，IBM 提供了所謂的「模塊化設計配接器」（plug-compatible）的升級路徑。客戶可以先從最基本的卡片系統開始——一台 1401 主機加上卡片讀卡機與印表機——處理傳統的會計工作。隨著業務增長，可以逐步添加磁帶機、磁碟機、遠端終端機等設備，而無需更換主機。這種漸進式升級的策略降低了客戶的決策門檻。

IBM 1401 的軟體生態也在 1961 年迅速發展。IBM 提供了 SPS（Symbolic Programming System）組譯器，這是 1401 的主要開發工具。SPS 使用符號位址與助憶碼來編寫程式，比直接用機器碼編程高效得多。IBM 還提供了 RPG（Report Program Generator）——一種專為產生商業報表設計的高階語言，使用 RPG 可以在數小時內完成原本需要數週的手工報表設計工作。

FARGO（Fourteen-o-one Automatic Report Generation Operation）是另一種針對 1401 開發的報表產生工具，比 RPG 更早推出，但兩者的目標一致：讓沒有深厚技術背景的商業人員也能自行開發資料處理程式。此外，IBM 還提供了 COBOL 編譯器——雖然受限於 1401 有限的記憶體容量，COBOL 程式的長度受到嚴格限制——仍然讓部分客戶能夠用標準化的商業語言開發應用。

1401 在 1961 年的快速普及對整個電腦產業產生了結構性的影響。它首次證明了電腦可以服務於廣泛的中型企業市場——在此之前，電腦幾乎完全由大型企業、政府與大學所壟斷。IBM 1401 的客戶群涵蓋了保險公司、銀行、製造商、零售商、醫院、學校與地方政府，使得「電腦化」從少數機構的特權變成了多數機構的選項。

IBM 1401 的成功也為 IBM 帶來了巨大的市場獨占力量。到 1960 年代中期，IBM 在商用電腦市場的佔有率超過了 70%，引起了美國司法部的反壟斷關注。1964 年 IBM 推出 System/360 時，刻意保持了與 1401 的（有限的）相容性，以便客戶可以順暢地從 1401 遷移到新的平台——這種「向上兼容」的策略成為此後電腦產業的標準做法。
