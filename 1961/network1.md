# Project MAC 提案：ARPA 資助的起點

1961 年是電腦網路發展史中觀念萌芽的年份。最重要的標誌性事件是 MIT 向美國國防部先進研究計畫局（ARPA）提交了 Project MAC（Multiple-Access Computer / Machine-Aided Cognition）的正式提案，獲得了 ARPA 的經費支持。這個專案直接導致了 MIT 在分時系統與電腦網路領域的長期主導。

Project MAC 的概念直接源於 J.C.R. Licklider 在 1960 年發表的論文〈Man-Computer Symbiosis〉。Licklider——一位心理學家轉型的電腦科學家——在論文中提出了一個深刻的見解：電腦的最終價值不在於取代人類，而在於與人類形成一種「共生關係」——人類負責直覺、創造與決策，電腦負責計算、記憶與資訊檢索，兩者透過高效的互動介面緊密協作。Licklider 認為，實現這種共生關係的關鍵技術是分時系統：讓多個使用者可以同時透過終端機與同一台電腦互動。

1961 年，Licklider 的設想找到了實際的行動方案。MIT 的 Robert Fano 與 Fernando Corbato 聯合提交了 Project MAC 提案，請求 ARPA 提供經費建造一套大規模的分時系統。ARPA 在 Jack Ruina 領導下批准了這個提案——這是一個決定性的政策選擇，它打開了 ARPA 大規模資助電腦科學研究的歷史大門。

Project MAC 的經費規模在當時是極為可觀的：ARPA 在數年間提供了約 200 萬美元的資金（約當今日 1,700 萬美元），用於購買電腦設備、僱用研究人員與支付營運費用。這筆資金使 MIT 能夠購買一台強大的 IBM 7094 電腦——配備了 32K 字記憶體與多個磁鼓——用於 CTSS 的後續開發與營運。

Project MAC 的分時系統——基於 Corbato 的 CTSS ——允許數十個使用者同時透過電傳打字機終端機與 IBM 7094 電腦進行互動。使用者的終端機透過專用線路或電話數據機連接到電腦，每個使用者獲得一個獨立的「視窗」——每秒鐘可以獲得約 0.1 秒的處理器時間。雖然這與今日的電腦使用體驗相比是極其粗糙的，但對於 1961 年的世界來說，這是革命性的。

Project MAC 在電腦網路發展中的意義不應被低估。ARPANET 的設計者之一 J.C.R. Licklider 在 1962 年被 ARPA 任命為資訊處理技術局（IPTO）的局長。Licklider 在 IPTO 的任期內（1962-1964）大力資助了多個大學的分時系統研究——包括 MIT 的 Project MAC、史丹佛的 SAIL 計畫、加州大學柏克萊分校的 Genie 系統——這些專案使用的電腦最終成為了 ARPANET 最早的節點。

Project MAC 還引入了一個重要的概念——「電腦公用事業」（Computer Utility），這個概念由 MIT 的資深電腦科學家 John McCarthy 在 1961 年的一次演講中提出。McCarthy 設想，未來電腦將像水電一樣成為公用事業——使用者只需支付使用費，即可透過終端機隨時隨地獲得運算能力。這個概念直接指向了雲端運算的願景——雖然這個願景花了五十年才真正實現。

Project MAC 的更深層影響在於它展示了政府研究經費對電腦科學發展的巨大推動作用。ARPA 在 1960 年代的投資模式——選擇頂尖的大學、提供長期穩定的經費、不干涉具體的研究方向——被證明是極為成功的。ARPANET 技術的後續發展，在很大程度上得益於 Project MAC 模式建立的研究基礎設施與人才儲備。

在 1961 年的當下，Project MAC 只是一個剛獲得經費批准的提案，但它所代表的理念——分時運算、人機共生、政府資助的研究——在接下來的十年中徹底改變了電腦科學的面貌。從 Project MAC 到 Multics，從 Multics 到 Unix，從 Unix 到網際網路，這條脈絡貫穿了整個 1960 至 1990 年代的資訊科技發展史。
