# 歷史穿梭：1968 年的種子與未來的森林

## 從遠端展示到網際網路

1968 年 Engelbart 的展示中使用遠端鏈路連接 SRI 和舊金山會場，從技術角度看只是一個有線電視會議。但這個展示預示了：

**1968 Engelbart 展示** → **1970s Xerox PARC Alto** → **1984 Apple Macintosh** → **1990s World Wide Web** → **2000s 社交媒體**

Engelbart 展示的滑鼠在 1980 年代成為標準配備；他展示的超文本鏈結在 Tim Berners-Lee 手中變成了 Web；他展示的即時協同編輯在 Google Docs 中成為日常工具。從 1968 年到今天的每一項人機互動創新，都可以追溯到舊金山的那個下午。

## Intel 處理器之路

Intel 在 1968 年的創立，開啟了半導體領域最傳奇的企業故事：

**Intel 1101 SRAM (1969)** → **1103 DRAM (1970)** → **4004 微處理器 (1971)** → **8080 (1974)** → **x86 架構 (1978)** → **Pentium (1993)** → **Core (2006)**

摩爾定律——由 Intel 共同創辦人 Gordon Moore 在 1965 年提出的觀察——在接下來的半個世紀中被證明驚人地準確。晶片上的電晶體數量從 1968 年 Intel 1101 的約 200 個，成長到 2020 年代高階處理器的數百億個。

## 結構化程式設計的分支

Dijkstra 的 1968 年短文引發了程式設計方法論的革命：

**結構化程式設計 (1968)** → **抽象資料型態 (1970s)** → **物件導向程式設計 (1980s)** → **設計模式 (1990s)** → **敏捷開發 (2000s)**

這個序列顯示了一個持續的趨勢：程式設計越來越關注「如何組織程式碼」而非「如何操作硬體」。每一層抽象都讓程式設計師能夠處理更複雜的問題。有趣的是，Dijkstra 對 goto 的批評在 2020 年代的函數式程式設計復興中獲得了新的共鳴——不可變資料和純函數被認為是更高層次的結構化。

## 資料庫的進化：從 IMS 到 NoSQL

IBM IMS 在 1968 年的部署標誌著資料庫管理系統的誕生：

**IMS 階層式資料庫 (1968)** → **CODASYL 網狀資料庫 (1971)** → **System R 關聯式資料庫 (1974)** → **Oracle (1979)** → **PostgreSQL (1996)** → **NoSQL (2000s)**

IMS 的階層式模型後來在 XML 資料庫和 JSON 文件儲存中獲得了新的生命。而 IMS 對事務處理的支援——ACID 屬性——成為了所有後續資料庫系統的基礎。

## 錯誤修正碼的宇宙

Hamming 在 1950 年發明的漢明碼，其影響遠遠超出了計算機科學：

**漢明碼 (1950)** → **Reed-Solomon 碼 (1960)** → **Viterbi 演算法 (1967)** → **LDPC 碼 (1963, 1990s 重新發現)** → **Turbo 碼 (1993)**

沒有錯誤修正碼，就不會有 CD/DVD、衛星電視、深空通訊。航海家號探測器傳回的木星和土星照片，在經過數十億公里的傳輸後仍然清晰——這是錯誤修正碼的功勞。Hamming 在 1940 年代被一個故障的讀卡機惹惱，結果改變了通訊技術的歷史。

## 1968 年的時代脈絡

1968 年是一個充滿動盪的年份：美國民權運動領袖 Martin Luther King Jr. 和 Robert Kennedy 被暗殺、越戰達到高峰、全世界學生運動風起雲湧。在這種背景下，電腦科學的發展顯得格外特別：

- **國防經費**：ARPANET 的研究經費來自冷戰時期美國國防部對分散式通訊網路的需求
- **太空競賽**：IMS 資料庫直接服務於阿波羅登月計畫
- **反文化運動**：Engelbart 的人機互動願景，與當時對個人解放和集體協作的追求相呼應
- **企業創新**：Intel 的創立模式——頂尖科學家離開大公司創立新創——成為矽谷的標準劇本

1968 年最迷人的地方在於：在今天看來理所當然的技術——滑鼠、超文本、資料庫、結構化程式、錯誤修正碼——都在這一年首次登上歷史舞台。
