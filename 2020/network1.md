# COVID-19 接觸追蹤 App：數位防疫的技術與隱私

## 疫情下的數位工具

隨著 COVID-19 在全球蔓延，各國政府和科技公司競相開發接觸追蹤應用程式，試圖以數位技術抑制病毒的傳播。這些 App 的核心原理是：利用智慧型手機的藍牙或 GPS 功能，記錄使用者之間的近距離接觸；當有人確診時，可以迅速通知可能與其接觸過的人。

## 隱私設計之爭

接觸追蹤 App 的設計圍繞一個核心矛盾：有效追蹤與個人隱私。

**集中式架構**：使用者的接觸記錄上傳到中央伺服器，由衛生當局進行比對和通知。這種設計有利於公共衛生分析，但隱私風險較高。英國的 NHS COVID-19 App 最初採用此架構。

**分散式架構**：比對過程在本地裝置上完成，不上傳使用者的位置或接觸記錄。Apple 和 Google 聯合推出的 Exposure Notification API 採用此架構。

## Apple/Google Exposure Notification API

2020 年 4 月，Apple 和 Google 出人意料地宣布合作，共同開發一個跨平台的接觸追蹤 API。這項合作在兩家公司的競爭歷史上是極為罕見的。

API 的設計原則：
- 不收集位置資料（僅使用藍牙信號強度估算距離）
- 使用滾動的隨機識別碼（匿名且定期更換）
- 資料僅儲存在本地，不上傳
- Apple 和 Google 不參與資料處理
- 當有確診時，使用者自願上傳自己的密鑰

## 全球實施案例

新加坡的 TraceTogether 是全球最早的接觸追蹤 App 之一。德國採用 Corona-Warn-App，基於 Apple/Google API 開發。台灣的「臺灣社交距離」App 於 2021 年上線。

接觸追蹤 App 的效果因國家而異。在亞洲國家（如韓國、台灣），由於公眾配合度較高和中央化的衛生系統，數位防疫取得了較好的效果。在歐美國家，隱私擔憂、安裝率不足和系統性問題限制了這些 App 的實際效用。
