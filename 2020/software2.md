# GitHub Copilot 的開發：AI 輔助程式設計

## 從 GPT-3 到程式碼生成

2020 年，GitHub（微軟子公司）開始探索將 OpenAI 的 GPT-3 模型應用於程式碼生成的可行性。研究的成果是 GitHub Copilot——一個基於 OpenAI Codex 模型的 AI 程式碼輔助工具。

Codex 是 GPT-3 的衍生模型，針對程式碼資料進行了專門的微調。它的訓練資料包含了數十億行的公開程式碼（主要來自 GitHub 上的開源專案），涵蓋了數十種程式語言。

## Copilot 的運作方式

Copilot 以 Visual Studio Code 擴充功能的形式提供。當開發者輸入程式碼時，Copilot 會根據上下文——檔案中已寫的程式碼、註釋和導入——即時建議接下來的程式碼。它不僅可以補全一行程式碼，還可以生成完整的函式、類別和測試。

Copilot 最令人印象深刻的能力是從註解生成程式碼：開發者只需要用自然語言描述想要的功能，Copilot 就可以生成對應的實作。

## 技術預覽與反響

2020 年，Copilot 仍處於內部開發階段，GitHub 在 2021 年 6 月才發布了技術預覽版。但在 2020 年，內部測試已經展現了驚人的潛力：對某些常見的程式設計任務，Copilot 的建議在超過 40% 的情況下被開發者直接接受。

## 爭議與挑戰

Copilot 也引發了多項爭議：

- **程式碼授權問題**：Copilot 在公開程式碼上訓練，但生成的程式碼是否應該遵守原始程式碼的開源授權？
- **抄襲風險**：Copilot 是否會逐字複製訓練資料中的程式碼？
- **開發者技能的未來**：AI 輔助編程是否會讓開發者失去對底層原理的理解？

這些問題在 Copilot 正式發布後繼續發酵，並在 2022 年引發了集體訴訟。
