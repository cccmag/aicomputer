# Short Code 語言 — 第一部電子計算機高階語言

一九五一年，UNIVAC I 上的 Short Code 語言仍然在持續演化。Short Code 由約翰·莫奇利（John Mauchly）於一九四九年開發，是世界上第一個為電子計算機設計的高階程式語言。

Short Code 的誕生源於莫奇利對程式設計效率的思考。在 UNIVAC I 的開發過程中，他眼睜睜看著程式設計師花費大量時間在機器語言的繁瑣細節上——計算絕對位址、追蹤記憶體分配、將數學公式分解為最基本的算術步驟。莫奇利認為，如果計算機本身能夠理解更抽象的指令，程式設計效率就會大幅提升。

Short Code 的語法設計反映了莫奇利的數學背景。程式可以包含類似代數表達式的結構——例如「X3 = 01 X1 + X2」，其中「01」是加法的代碼。Short Code 支援間接位址——一個變數可以存放另一個變數的位址——這個概念在高階語言中後來被稱為「指標」。

Short Code 的表面形式與今日的高階語言相去甚遠——它仍然使用數字代碼而非英文關鍵字。但它在概念層次上的提升是巨大的：程式設計師不再需要關心指令在記憶體中的具體位置，也不需要手動管理暫存器的使用。

Short Code 的直譯器本身是一段精妙的程式。它讀取每個符號指令，解析操作碼和運算元，然後生成對應的 UNIVAC I 機器碼序列，並立即執行。這種直譯式執行方式在當時是必要的——UNIVAC I 的記憶體容量根本不足以同時容納 Short Code 程式和它生成的完整機器碼版本。

Short Code 的使用者社群雖然很小，但其影響卻遠超預期。Short Code 的使用者——主要是 Remington Rand 的程式設計師和部分客戶——是第一批體驗到高階語言優勢的人。他們的經驗直接影響了後來的程式語言設計，特別是 FORTRAN（一九五四年由 IBM 的約翰·巴克斯領導開發）。

從 Short Code 到 FORTRAN 到 ALGOL 到 C 到 Java 到 Python，程式語言的演化是一條持續提升抽象層級的旅程。一九五一年的 Short Code 是這條旅程上第一個有意義的里程碑。它的創作者莫奇利可能無法預見後來的軟體產業規模，但他已經抓住了程式語言設計的核心矛盾：人類的思考方式與機器的執行方式在本質上是不同的，語言設計者的任務就是在兩者之間找到最佳平衡點。
