# SQL Slammer 蠕蟲：網際網路的教訓

## 十分鐘的災難

2003 年 1 月 25 日清晨 5:30 UTC，SQL Slammer 蠕蟲開始在網際網路上擴散。這隻僅 376 位元組的蠕蟲利用了 Microsoft SQL Server 2000 的緩衝區溢位漏洞（CVE-2002-0649），透過 UDP 1434 埠進行隨機 IP 掃描感染。

Slammer 的傳播速度前所未見：它在感染啟動後 3 分鐘內就達到每秒 5,500 萬次掃描的全速，10 分鐘內感染了超過 75,000 台伺服器。相較之下，2001 年的 Code Red 蠕蟲需要 37 分鐘才能翻倍感染數量，Slammer 只需要 8.5 秒。

## 全球影響

Slammer 雖然沒有攜帶惡意負載，但它產生的巨大流量導致了廣泛的附帶損害：
- 南韓的網路服務大規模中斷
- 美國航空的航班延誤（後端系統癱瘓）
- 芬蘭的 ATM 提款機故障
- 數千台路由器因路由表更新泛洪而崩潰

諷刺的是，Microsoft 早在 2002 年 7 月就發布了 SQL Server 2000 的漏洞修補程式（MS02-039），但多數系統管理員尚未部署。

## 歷史意義

SQL Slammer 是第一個「超快速蠕蟲」的實際展示——安全研究人員 Staniford、Paxson 和 Weaver 在理論上預測了這種可能性，Slammer 將其變為現實。它促使企業和組織加速修補管理流程，並催生了更快的入侵偵測和響應機制。Slammer 的教訓很明確：在網際網路時代，漏洞修補的延遲是以秒計算的。
