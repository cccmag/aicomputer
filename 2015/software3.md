# Kubernetes 1.0：容器編排的新標準

## 從 Google 內部到開源社群

2015 年 7 月 21 日，Google 正式發布 Kubernetes 1.0，並宣布將專案捐贈給新成立的 Cloud Native Computing Foundation（CNCF）。這標誌著容器編排技術從 Google 內部專案（Borg 和 Omega）走向開放治理的關鍵一步。

Kubernetes（希臘文意為「舵手」或「領航員」）的設計繼承了 Google 十五年來在大規模容器管理上的經驗。它的核心抽象包括 Pod、Service、ReplicaSet、Deployment 等，形成了一套靈活而強大的容器宣告式管理系統。

## 生態效應

Kubernetes 1.0 的發布引發了容器生態系統的重新洗牌。Red Hat、IBM、VMware 等主要企業迅速投入 Kubernetes 生態。Microsoft 在 Azure 上推出了 Kubernetes 容器服務，AWS 也開始支援 Kubernetes。

到 2015 年底，Kubernetes 已成為容器編排領域的領先專案。其 API 設計——宣告式規格（desired state）加上控制迴路（control loop）——成為整個雲端運算領域的新範式。

## CNCF 的成立

CNCF 的創立對於容器技術的發展至關重要。作為一個中立的開源基金會，CNCF 確保了 Kubernetes 不會被單一供應商鎖定，並建立了一套開源治理的最佳實踐。這種治理模式後來被廣泛複製。
